## 3. 运行主链 `run_cycle`（每轮）

```
GetCommand → _dispatch_command(cmd_router：解析+幂等+命令账本)
  → receive_signal(按 SIGNAL_SOURCE；总线不可用→禁新开、不阻断持仓)
  → gate_summary(5 门 + 运行时急停覆盖)
  → 分支(优先级)：
       recovery=RECOVERY_BLOCKED → 禁新开
       有持仓 → manage_cycle（急停下仍运行）
       急停 → KILLED
       已锁定 → _attempt_commit（预提交13门 → **开仓活动 entry_campaign**：信用底线内 maker、保护腿先成交、跨轮持久 → 两腿达标冻结入场快照 / 超额放弃回退）
       信号不放行/恢复未就绪 → WAIT_SIGNAL / OFFLINE_MANUAL
       否则 → 维护垂直推荐库(节流 PLAN_REFRESH_SECONDS) + 短确认码 → HARD_APPROVAL_WAIT
  → 渲染交互控制台(置顶) + 操作提示 → _emit(LogStatus+Log)
```

阶段：`WAIT_SIGNAL / OFFLINE_MANUAL / RECOMMEND_READY / HARD_APPROVAL_WAIT / PLAN_LOCKED / POSITION_MANAGE / KILLED / RECOVERY_BLOCKED`。

## 4. 交互层

- **状态栏交互控制台**（每轮置顶）：阶段 / 执行门控(进/退/对冲开/对冲减 ✓✗) / 信号接收 / 待批方案+短确认码 / 预提交 / 开仓 / 风险动作 / 软授权(状态+授权码) / 止盈资格 / 退出活动 / 对冲 / **操作提示**。
- **FMZ 按钮**（机器人「交互」区配置；`cmd_router` 解析 `名:参数`）：

  | 按钮 | 类型 | 参数 | 命令类型 | 作用 |
  |---|---|---|---|---|
  | `执行` | 字符串 | 方案确认码 | EXECUTE(消费型) | 锁定对应方案→预提交 |
  | `拒绝` | 按钮 | — | REJECT | 放弃待批 |
  | `授权止盈` | 字符串 | 持仓授权码 | EXIT_AUTHORIZE(消费型) | 允许自动止盈退出 |
  | `撤销授权` | 按钮 | — | EXIT_REVOKE | 撤销退出授权 |
  | `风险退出授权` | 字符串 | 持仓授权码 | RISK_EXIT_AUTHORIZE(消费型) | 独立风险退出预算授权 |
  | `急停` | 按钮 | — | KILL | 停新风险、撤开仓挂单 |
  | `恢复` | 按钮 | — | RESUME | 清急停+清锁，要求新硬批准 |

- **短确认码**：面板逐方案显示 4~6 位活码；内部按 `session+strategy+quality+plan_hash` 全约束匹配；质量按 S:PM 释放 0.1 分桶 → 子桶波动不改码（输入期间稳定），材料级变化才换码、旧码自动过期。
- **幂等**：键 `robot_id+session_id+refresh_seq+command_type+nonce`；消费型一次性消费（防跨轮/重启重复下单），切换型可重复；全部入命令账本。

## 5. 门控模型（5 门 + 对冲细分）

`gate_decision(action, allow_entry, allow_exit, allow_hedge, kill_new_risk, emergency_reduce_only)`：

| 动作 | 允许条件 | reduce_only |
|---|---|---|
| ENTRY | allow_entry ∧ ¬kill ∧ ¬emergency | False |
| EXIT | allow_exit（降风险，不被 kill/emergency 阻断） | False |
| HEDGE_OPEN | allow_hedge ∧ ¬emergency（kill 不阻断：对冲压尾部=降险） | False |
| HEDGE_REDUCE | allow_hedge（kill/emergency 下仍允许） | **True** |

急停=停新风险+撤开仓单，但退出/对冲减仓/对账/孤儿清理继续；恢复=重新对账+要求新计划硬批准。

## 6. 对冲场所（可选；§F1）

`HEDGE_VENUE` 操作者显式选择（**非运行时自动切换**）：
- **DERIBIT**（默认）：BTC-PERPETUAL，反向(USD 合约)，与期权同所便于统一对账/恢复；对冲开仓可 taker。
- **BINANCE**：BTCUSDC 永续，线性(BTC)，**USDC maker 0 费** → 强制 maker(post-only)；对冲腿非高频、等成交省成本。
- 目标换算：DERIBIT `delta_btc·spot/contract_size`；BINANCE `delta_btc`（线性）。
- **取舍（重要）**：选 Binance 时对冲在**异所异账户**，失去 Deribit 同所统一对账/保证金/恢复的便利；孤儿/恢复检查须读所选场所的 perp 持仓；跨所失配恢复更复杂。真实币安下单调用形态须真实机器人确认（默认门关不触发）。
