## 0. 执行摘要

前置信号层在某个时点给出信号后，**该信号"论证前提"的存活时长，强依赖它诞生在哪个时区**。一个在亚洲薄盘里成立的偏空信号，到欧/美盘携资金与时事冲入时，可能被推向更有利或更不利的方向——**无论哪种，都与"信号成立时的微结构前提"相悖**。

本文主张：把 UTC+8 的 24 小时按"信号前提耐久度"分为**高 / 中 / 低**三档，并为每档给出市场微结构机制 + 客观调查证据。关键定性:

- 它**不是**证据质量置信（`decision.confidence`，语义 `EVIDENCE_QUALITY_NOT_WIN_RATE`），而是一个**正交的"前提耐久度 / 时间稳健性"先验**——衡量"在外源性流动性/事件浪潮到来前，本信号的论证前提有多大概率原样存活"。
- 它由**两条底层风险轴**决定：①**形成时的流动性深度**（证据是建在真实成交还是薄盘上）；②**临近外源性冲击暴露**（多快会有美盘/欧盘/定时数据的浪潮可能覆盖前提）。
- 它是**确定性本地层**，初版**只打标、不改 confidence**（影子模式），校准后再决定是否做软门控；在用本策略历史数据校准前，标 `UNCALIBRATED`（沿用全工程的延迟标定纪律）。

## 1. 前置认知:为什么"出信号的时区"会动摇论证前提

### 1.1 问题本质:点估计前提 vs 即将到来的体制浪潮

前置信号层的论证（量价、CVD、宏观、偏斜、Gamma…）本质是对**当下微结构截面**的一次加权判断。它隐含一个前提:**在信号被消费的时间窗内，微结构大体稳定**。但加密市场的微结构有强烈的**时区节律**——薄盘时段成立的信号，前提随时可能被下一波主导流动性 / 定时事件**整体重写**。

> 你的核心观察是对的:**临近美盘的信号尤其脆弱**——不是因为它"错"，而是因为它坐在一个**即将开闸的活板门**上，美盘流动性或外源性事件可以把它推向任一方向，而这与"信号成立时的前提"在原则上相悖。

### 1.2 两条底层轴(本分类的理论骨架)

| 轴 | 含义 | 高风险表现 |
|---|---|---|
| **A. 形成时流动性深度** | 证据是建在真实成交流上，还是薄盘易被插针上 | 薄盘 → 证据脆、易被小单扫翻 |
| **B. 临近外源性冲击暴露** | 多快会有美/欧盘或定时数据浪潮可能覆盖前提 | 浪潮在前方且临近 → 前提"待覆盖"风险高 |

三档置信区 = 这两轴的合成。**注意两者是不同风险**:亚洲午后是"A 高"(薄盘)，美盘前是"B 高"(临近催化剂)——它们都该判低，但成因不同。把两轴讲清楚，才不会把"低流动性"和"高事件风险"混为一谈。

### 1.3 与 `decision.confidence` 严格区分

`confidence` 是**证据质量**(横截面有多强)。本层是**时间稳健性**(这份证据的前提能活多久)。二者正交:一个证据极强的信号若诞生在美盘前活板门上，仍可能被一条 CPI 打穿。**绝不可把时区档位悄悄乘进 confidence**——那会污染既有语义、且把两种风险双重计数。正确做法见 §6.3。

## 2. 客观经验与调查证据

加密资产"依附美盘/关键流动性时段"已是可量化的事实，这正是时区分类在 2026 年比以往更重要的原因:

1. **美盘主导流动性、非美时段创新低**:Kaiko 研究显示，随美国参与度上升，**美国时段以外的交易量已降至历史低位**，USD 市场在**亚洲与欧洲时段出现明显流动性缺口**；币安平均小时成交在**美股开盘附近见顶**，Coinbase 则在**美股收盘前 3–4pm 纽约的 ETF NAV 定盘窗**放量。([Kaiko][k1])
2. **加密随美股宏观体制共振**:BTC 与标普 500 的相关性 2022 年后大幅上升，2026 年初 30 日滚动相关达约 **0.74**，部分日内窗口 r² 高至 **0.94**。([Phemex 综述][p1]) → 美国数据/开盘是加密最主要的**外源性冲击来源**。
3. **周末更薄、机构离场**:周末成交量较工作日**下降约 20–40%**，点差走阔、小单即可造成更剧烈的价格移动，机构要到周一才回桌。**注意时效**:CME 已于 **2026-05-29 将 BTC/ETH 期货转为 7×24 交易**，经典"周日 23:00 UTC 重开的 CME 跳空"已基本消失；但**周末薄盘与价格发现集中于美盘工作时段的结构依旧**。([Phemex][p2]，[CoinDesk][cd1])
4. **日内周期性与股市挂钩**:学术研究记录了 BTC 成交量/波动率的**日内周期性，且与股市时段相关**；做市深度也呈现明显的日内节律。([ScienceDirect][sd1]，[Amberdata][am1])

> 定位:以上是**已被广泛观察/研究的市场结构事实**，作为本分类的**经验先验**。但**各档的具体边界仍须用本策略自身的 `signal_review.jsonl` 历史回测验证**（§7）——这才是最贴合的"调查"。

## 4. 每档为何是这个置信度(逐区理论依据)

### 4.1 高置信区 · 23:00–04:00（美盘深流动性 → 收盘）

- **机制**:这是全球加密**最深流动性**窗（Kaiko:美盘主导；ETF NAV 03:00–04:00 放量）。当日美国主要定时催化剂（08:30ET 数据=20:30、FOMC 02:00）此时**多已落地并被消化**。
- **为何高**:两轴同时最优——证据建在**真实、最具信息量的机构流**上(轴A深)，且**最大的外源性浪潮已经来过而非临近**(轴B低)。信号前提的"待覆盖"风险最小，故最耐久。
- **例外(交日历叠加层处理)**:FOMC 决议/发布会(02:00–02:30) 当晚此区反而是事件中心 → 由 §5.3 黑名单覆盖为低。

### 4.2 低置信区 · 18:00–21:30（美盘前·数据跑道）——你的主张，成立

- **机制**:流动性虽在上升，但此窗**正压在美国数据(20:30)与纽约开盘(21:30)的正上方**。加密与美股 r² 可达 0.94，意味着一条 CPI/NFP 或开盘流动性就能**整体重写**此前在欧/亚盘成立的前提。
- **为何低**:这是**轴B(临近冲击)驱动的低**，不是轴A。信号本身可能证据不弱，但它**坐在活板门上**——把它列入低置信、要求"等美盘开后再确认"，与"防前提被外源性事件冲击"的原则完全一致。

### 4.3 低置信区 · 11:30–15:00（亚洲午后→欧洲前）

- **机制**:亚洲收力、欧洲未开，是**记录在案的流动性低谷**（非美时段缺口）。
- **为何低**:**轴A(薄盘)与轴B(欧美都还在前面)同时高**——证据建在薄盘上，且最大的两波浪潮全在前方，"待覆盖"暴露最大。

### 4.4 低置信区 · 04:00–08:00（美盘收后空档）

- **机制**:美盘已收、亚洲未起，**全天最薄盘口**，小单即可插针/跳空（周末尤甚）。
- **为何低**:**轴A 最差**——信息含量最低、最易被操纵；此处成立的信号最不可信。

### 4.5 中置信区 · 08:00–11:30 / 15:00–18:00（亚洲早盘 / 伦敦早盘）

- **机制**:各自有**真实区域流动性**(东京+港中A;伦敦开盘)，价格发现回升。
- **为何中**:轴A 尚可，但**轴B 仍偏高**——欧美主浪潮都还在数小时后，前提仍有被覆盖的余地。给"中"，提示"方向可参考，但留意后续时段冲击"。

### 4.6 中置信区 · 21:30–23:00（纽约开盘动荡）

- **机制**:流动性最深，但**开盘与数据后的双向波动剧烈**，止损猎杀常见。
- **为何中**:轴A 极优但**开盘湍流**会洗掉过早成立的信号 → 不升至高，待 23:00 后趋稳再算高。

## 5. 动态叠加层(覆盖静态表的修正项)

静态时钟表只刻画"结构性"流动性节律；以下叠加层处理"情景性"催化剂，**优先级高于静态表**。

### 5.1 夏令时(必须正确处理)

美/欧锚点随其本地夏令时移动。**冬令时(美 EST=UTC−5，伦敦 GMT=UTC+0)下，所有美国相关边界在 UTC+8 上整体后移 1 小时**:纽约开盘→22:30、数据→21:30、收盘→05:00、伦敦开盘→16:00。
→ **实现不要硬编码 UTC+8 时钟**，应由"纽约/伦敦当地时间 + 其 DST 规则"动态推导会话状态，再映射回 UTC+8。静态表仅作 EDT 参照。

### 5.2 周末修正

周六、周日（及周五美盘收后至周一机构回桌前）：成交量 −20~40%、更薄、小单更易致剧烈移动 → **整体下调一档**(高→中、中→低、低维持低)。CME 已 7×24，**不再建模周日重开跳空**，但周末薄盘与"价格发现集中于美盘工作日"的结构仍在。

### 5.3 定时事件黑名单(最关键的情景覆盖)

已知高影响释放（**CPI / NFP / FOMC / 大额期权到期 OPEX / 重大监管或宏观事件**）前后 N 小时，**无视时钟一律置低**。这是日历驱动的硬覆盖，**直接对应你担心的"外源性事件冲击"**。N 可分级:发布前 2h、发布后 1h 收敛期。

### 5.4 资金费率结算(次要)

加密永续在 00:00/08:00/16:00 UTC（= 08:00/16:00/00:00 UTC+8）结算，常伴小幅波动尖峰 → 作弱提示，不单独改档。

### 5.5 边界缓冲与滞后/前置:为什么不是"整体后移"

**批评成立的部分**:§3 表把档位边界画成了**时钟瞬切**（09:30 一到就翻档）。现实中会话/事件的影响**不在边界瞬间显现**——有建仓与人类/算法消化的**滞后(lag)**，跨边界的**持续(persistence)拖尾**。硬边界确实过于理想化。

**但"整体后移所有区"是错误的修法**:影响不是纯滞后，而是对理想边界的**双向涂抹(lead + lag)**，且本表两半行为相反——

- **前置(lead)型风险区**:**美盘前活板门(LOW)** 本质是**对尚未到来的浪潮的预期风险**——建仓、盘前期货、数据前定位常在时钟事件**之前**就发生，危险**领先**时钟。**若把它后移，等于把保护挪到美盘开盘之后；而那时浪潮已到、消化后反而转高——方向完全反了。**
- **滞后(lag)型耐久区**:**美盘深流动性 HIGH** 是"催化剂已消化、价格发现稳定"的状态，它**滞后**于开盘才成立。这里你的直觉对——**HIGH 的起点应推后**，让它在湍流与消化结束后"挣得"进入。
- **持续(persistence)**:收盘后漂移、事件后收敛使低档/事件影响**向后拖尾**。

> 结论:正确修正是**非对称 + 边界模糊**，不是刚性平移——**推后 HIGH 起点(滞后/消化)、保留甚至前扩领先型风险区(前置)**。净效果是"在边界附近更久地停留在低档"，方向上符合你"别太快给高"的直觉，但机制上不是"整体后移"。

**落地办法(两条，零新依赖):**
1. **缓冲带(boundary buffer)**:每个边界两侧各设 ~30–60 分钟过渡带，带内取**相邻两档中更保守者**(或标 `TRANSITION`)。一个参数 `boundary_buffer_min` 即可，约 +10 行。
2. **非对称偏置**:HIGH 起点额外后推一个消化窗(如 NY 开盘后 +60~90 分钟才算 HIGH)；领先型 LOW（美盘前）起点不动或前移。

**关键纪律:别用"新的精确数字"替换"旧的精确数字"——那只是把虚假精度搬个位置。** 每个边界真实的领先/滞后量应由 §7 在本策略历史上**实测**:把每条信号对齐到最近的会话/事件边界，观察"前提存活率相对时钟边界在哪个偏移处才真正改善"，据此定 buffer 宽度与 HIGH 滞后量。**校准前，缓冲带从宽、就低不就高。**

## 6. 落地设计(确定性本地层)

### 6.1 形态:纯函数，可复现、可单测

```
classify_session_zone(ts_utc, calendar) -> {
  zone: "HIGH"|"MEDIUM"|"LOW",
  premise_durability: 同 zone,
  liquidity_depth: "DEEP"|"MODERATE"|"THIN",   // 轴A
  catalyst_exposure: "LOW"|"MEDIUM"|"HIGH",     // 轴B
  rationale_code: "US_DEEP_POST_CATALYST"|"PRE_US_TRAPDOOR"|"ASIA_AFTERNOON_LULL"|...,
  rationale_cn: "...",
  dst_mode: "EDT"|"EST",
  is_weekend: bool,
  event_blackout: {active: bool, event: "CPI"|"FOMC"|..., phase: "PRE"|"POST"} | null,
  calibration_state: "UNCALIBRATED"
}
```

- **输入**:信号确认时戳(UTC) + 一个日历对象（DST 规则 + 高影响事件表）。
- **位置**:策略代码内，在**信号成立后、构卡时**调用；**不进 LLM 请求**。
- **运算顺序**:静态时钟表 → 周末修正(§5.2) → 事件黑名单覆盖(§5.3) → 输出。

### 6.2 写回卡片

挂到 `signal_window.session_context`（与 `decision` 平级隔离，不进 `decision`）:

```json
"signal_window": {
  "...": "...",
  "session_context": {
    "zone": "LOW",
    "liquidity_depth": "MODERATE",
    "catalyst_exposure": "HIGH",
    "rationale_code": "PRE_US_TRAPDOOR",
    "rationale_cn": "美盘前数据跑道：信号前提易被美国数据/开盘流动性整体重写",
    "dst_mode": "EDT", "is_weekend": false, "event_blackout": null,
    "calibration_state": "UNCALIBRATED"
  }
}
```

### 6.3 语义与作用方式(分阶段，守住既有 confidence 语义)

- **Phase 0 · 影子打标**:只写 `session_context`，**不改 confidence、不改门控**。同时落库供 §7 校验。
- **Phase 1 · 运营提示**:前端/推送展示档位与 `rationale_cn`，作为人工"前提耐久度"提示。
- **Phase 2 · 软门控(仅校准后)**:若回测证明 LOW 区前提失效率显著更高，可让"LOW 区 + 本就弱支持"的信号**升一级谨慎 / 要求确认**，但**仍不乘进 confidence**——而是作为独立的 `temporal_gate`。
- **红线**:本层永不**单独**否决一个强信号，也不**悄悄**改写证据质量；它是正交的耐久度维度。

### 6.4 架构归属:进策略本体(引擎)，不作串行下游

**结论:分类引擎进策略本体，在构卡时算进基础卡;事件日历外置为 config 数据。** 不要把它当成像 LLM 复核或前端那样的"串行下游业务"。

理由——把 LLM 层 / 前端推到下游的每一条性质，对时区分类**全部反转**:

| 属性 | LLM 复核层 | 前端 | **时区分类** |
|---|---|---|---|
| 确定性 | 否(随机) | — | **是(纯函数)** |
| 外部依赖 | API/网络/密钥 | 浏览器 | **无** |
| 失败面 | 高(超时/坏 JSON) | — | **几乎为零** |
| 运行成本 | 按 token 计费 | — | **零** |
| 影响信号语义/门控 | 否(只读建议) | 否(展示) | **是(前提耐久度;Phase 2 可软门控)** |
| 需进可复现哈希卡 | 否(富化追加) | 否 | **是(信号内在语义)** |

四点决定性论据:
1. **它属于信号的内在语义**:"这份证据能撑到几点"和方向、置信一样，是信号本身的属性，应在**基础卡(被 `integrity` 哈希、可复现)**里，而非事后附加。
2. **算一次、在源头算**:本体构卡时已持有 `confirmed_at`(UTC)，分类零新增数据依赖。若放下游，物化器/前端要**各自重实现 DST + 事件日历**→ 双份真相、易不同步。
3. **门控前置条件**:Phase 2 若做 `temporal_gate`(LOW 区 + 弱支持→升谨慎)，那是**决策层**行为，下游无权做;放下游 = 迟早被迫迁回本体。
4. **零失败面**:它没有 LLM 的随机/超时/计费，没有任何理由享受"软失败旁路"待遇。

**唯一支持下游的真实论点 = 事件日历每月变(CPI/NFP/FOMC 日期)，不想为此重部署策略。** 化解办法:**逻辑进本体、日历作数据**——分类引擎(稳定逻辑)进本体，事件黑名单(易变数据)作为外置 config/JSON 加载;DST 由 `zoneinfo` 按算法自动判定，无需维护。

> 与"不进 LLM 请求"不冲突:本层在**本体计算并写入卡片**;是否把 `session_context` 转发进 LLM 包是另一回事，默认**不转发**(保持确定性时间先验与 LLM 定性判断的清晰分工)。

### 6.5 代码量评估(Python，足量功效)

**本体引擎约 80–130 行**即可达足量功效(静态表 + `zoneinfo` 处理 DST + 周末降档 + 事件黑名单覆盖 + 输出结构 + 构卡接线);事件日历作为外置数据**另计**。下面骨架用以支撑该估算(`zoneinfo` 让 DST 在本地时间维度自动正确，是省代码的关键):

```python
from zoneinfo import ZoneInfo
_NY, _LON, _CN = ZoneInfo("America/New_York"), ZoneInfo("Europe/London"), ZoneInfo("Asia/Shanghai")

_RATIONALE_CN = {
    "US_DEEP_POST_CATALYST": "美盘深流动性、催化剂已消化，前提最耐久",
    "US_OPEN_TURBULENCE":    "纽约开盘湍流，流动性足但易被双向洗",
    "PRE_US_TRAPDOOR":       "美盘前数据跑道，前提易被美国数据/开盘整体重写",
    "POST_US_DEADZONE":      "美盘收后空档，盘口最薄、易插针",
    "LONDON_EARLY":          "欧洲早盘，流动性补位但美盘仍在前方",
    "ASIA_MORNING":          "亚洲活跃早盘，区域流动性真实但欧美仍在前方",
    "ASIA_AFTERNOON_LULL":   "亚洲午后低谷，薄盘且欧美都在前面",
    "EVENT_BLACKOUT":        "定时高影响事件窗口，强制低置信",
}
_DOWN = {"HIGH": "MEDIUM", "MEDIUM": "LOW", "LOW": "LOW"}

def _base_tier(ts):                       # 以纽约本地时间为主轴，DST 自动正确
    nh = ts.astimezone(_NY).hour + ts.astimezone(_NY).minute / 60.0
    lh = ts.astimezone(_LON).hour
    ch = ts.astimezone(_CN).hour + ts.astimezone(_CN).minute / 60.0
    if 11.0 <= nh < 16.0: return "HIGH",   "US_DEEP_POST_CATALYST"
    if  9.5 <= nh < 11.0: return "MEDIUM", "US_OPEN_TURBULENCE"
    if  6.0 <= nh <  9.5: return "LOW",    "PRE_US_TRAPDOOR"
    if 16.0 <= nh < 20.0: return "LOW",    "POST_US_DEADZONE"
    if  8   <= lh < 11:   return "MEDIUM", "LONDON_EARLY"
    if  8.0 <= ch < 11.5: return "MEDIUM", "ASIA_MORNING"
    return "LOW", "ASIA_AFTERNOON_LULL"

def classify_session_zone(ts_utc, event_calendar=None):
    ev = _event_blackout(ts_utc, event_calendar)          # ~12 行: 遍历事件窗口判 in-range
    zone, code = ("LOW", "EVENT_BLACKOUT") if ev else _base_tier(ts_utc)
    is_weekend = ts_utc.astimezone(_CN).weekday() >= 5
    if is_weekend and not ev:
        zone = _DOWN[zone]
    return {
        "zone": zone, "rationale_code": code, "rationale_cn": _RATIONALE_CN[code],
        "is_weekend": is_weekend, "event_blackout": ev,
        "dst_mode": "EDT" if ts_utc.astimezone(_NY).dst() else "EST",
        "calibration_state": "UNCALIBRATED",
    }
```

| 分档 | 内容 | 估算行数(本体) |
|---|---|---|
| 最小可用 | 静态表 + DST + 周末 + 写卡（无事件黑名单） | **~60 行** |
| **足量功效** | ＋事件黑名单覆盖 ＋ `rationale_code/cn` ＋ 输出结构 ＋ 构卡接线 | **~80–130 行** |
| 加固 | ＋两轴字段(`liquidity_depth/catalyst_exposure`) ＋ 单测桩 | ＋~30–50 行(单测在 tests/) |
| 环境兜底 | 若 FMZ 缺 `tzdata`/`zoneinfo` 不可用 → 手写 US/EU DST 判定 | ＋~25–35 行 |

**值不值:决定性地值。** ~100 行纯确定性代码、零运行依赖、零失败面、在源头算一次、进可复现哈希卡、且为未来门控就位。下游方案**更贵**(物化器+前端重复 DST/日历逻辑、同步 bug)，且一旦影响门控就要迁回本体。事件日历外置后，月度更新不触碰策略逻辑。

## 7. 校准与验证(用本策略自身历史数据——真正的"调查")

经验先验给了**排序假设**:前提失效率 LOW > MEDIUM > HIGH。必须在自有数据上证伪:

1. **打标**:对 `signal_review.jsonl` 历史每条真实信号回填其形成 `zone`。
2. **度量前提存活**（每条信号在其后 K 小时 / 至下一主导时段内）:
   - **前提失效率**:方向前提被反向击穿的比例；
   - **不利偏移分布(adverse excursion)**:信号后最大不利位移；
   - **信号后已实现波动**:验证 LOW 区是否确实更"易被冲"。
3. **验证排序**:若各指标按 HIGH<MEDIUM<LOW 单调，则先验成立 → 用数据**重定边界**；否则**收缩或弃用**该层(别养无效层，同独立视角文档的退役纪律)。
4. 校准达标前 `calibration_state` 恒为 `UNCALIBRATED`。

> 主 KPI 不是"哪个时区方向更准"，而是"**时区是否预测前提存活**"。

## 8. 边界与陷阱

- **是先验，不是否决**:与既有数据质量门控（`quality`）勿双重计数；时区低≠数据缺失。
- **DST/节假日是真实 bug 面**:必须从会话日历动态推导，硬编码时钟半年会错 1 小时。
- **体制会漂移**:ETF 资金、CME 7×24、亚洲份额变化都会移动边界 → 边界须**周期性再校准**，不是一次定死。
- **别把它当 confidence**:守住 §1.3/§6.3 的正交性，否则污染既有语义。
- **与 LLM 层的分工**:本层是**确定性时间先验**，LLM 独立视角是**定性体制/尾部判断**——前者进卡、后者进复核，互不混入(本层明确不进 LLM 请求)。

---

## 9. 结论

- **可行且应做**:加密已实证"依附美盘流动性 + 随美股宏观共振"，时区对信号前提耐久度有**结构性影响**，用确定性本地层把它显式化，是低成本高价值的减偏。
- **正确定位**:它是与证据质量正交的**前提耐久度先验**，初版只打标、延迟标定，守住不污染 confidence 的红线。
- **你的两个主张都成立**:① 把 24 小时分高/中/低三档；② **临近美盘信号列入低置信**——因为加密与美盘 r² 可达 0.94，前提随时可能被美盘流动性或定时数据整体重写。
- **架构归属(§6.4/6.5)**:**引擎进策略本体、算进基础卡**,不作 LLM/前端那样的串行下游——它确定性、零依赖、零失败面、属信号内在语义、且是未来门控前置;**事件日历外置为 config**以避免月度更新触碰策略逻辑。足量功效约 **80–130 行 Python**。
- **边界不是硬切(§5.5)**:你"影响不会瞬间显现"的观察成立，但修正**不是整体后移**——影响是 lead+lag 双向涂抹，美盘前活板门是**前置**风险(后移会反向)、美盘深流动性是**滞后**状态(起点应后推)。改用**缓冲带 + 非对称偏置(就低不就高)**，精确偏移交 §7 实测，不手填新精度。
- **一句话**:**证据质量回答"信号有多强"，时区耐久度回答"这份强能撑到几点"——薄盘与临近活板门上诞生的信号，再强也要打折等确认。**
