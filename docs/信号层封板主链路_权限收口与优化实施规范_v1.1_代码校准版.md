## 0. 执行摘要（校准版）

当前信号层的理论框架值得保留：DIE + Anchor 识别价差窗口；TMV / CVD / Funding / Macro / SRD 提供方向与反证；GGR / net Gamma 识别结构稳定性；LLM 与人工负责模型风险审核。

**v1.0 的结论方向成立，但需要三处校准：**

1. **量级排序**：真正的结构失衡只有一处——**CVD 两窗合计方向权重 ≈1.47，已超过 TMV 的 1.00**（详见 §2.2 权重预算表）。Funding 双重计权、GGR 正向 boost、GGR 空间票都属于**小量级**越权。SRD 则是一个**0.70 权重的大方向输入**，降级它是高回归风险动作，不是清理。

2. **已满足项不要再"修"**：代码**已经**没有单一总置信、**已经**给 EDB 打了 `UNCALIBRATED` 标、增强 GEX 源**已经**是"只降级 regime 信任、从不投方向票"、负 Gamma 下 GGR **已经**是纯门（空间权重为 0）。这些是已具备的良好性质，应从"问题清单"移到"需保持的不变量"。

3. **证据强度不能夸大**：v1.0 §11 的合成模拟（方向 +3.33pct 等）是**在"三个流因子共同观测同一瞬时 shock"的生成假设下的代数后果**，对真实 BTC **零外部效力**。封板放行门必须是真实样本外 replay（§9 Phase B/C），不是模拟数字。

**本版仍不建议新增因子域。** 下一步先做高杠杆低风险两项（CVD 合并、Funding 去重），把 SRD/Macro 降级与"流权重封顶"作为**最高回归风险项**严格压在消融实验后面。


## 2. 发现的问题（按真实量级与确定性重排）

### 2.1 【实质·高杠杆】CVD 4h/12h 双票，合计权重压过 TMV

`_cvd_votes` 对 fast_4h / slow_12h 各产生一票，base 同为 0.70（:3026, :426）。两窗来自同一主动成交序列，合计最大有效权重 ≈1.47 **超过 TMV 的 1.00**。

**精确诊断（修正 v1.0）**：被重复计权抬高的**不是 coverage**——`_coverage`（:3224）在分子 `present` 和分母 `total` 里都把 CVD 计了两次（`2.0*CVD`，:3230），归一化后基本抵消。真正被抬高的是：

- **加权 score 的主导权**：流量以 1.47 的权重盖过趋势；
- **agreement 一致度**：两条高度相关的同向票各自计入 `_agreement`（:3212）的同号分子，把"一份流证据"伪装成"两份共识"。

### 2.2 【小量级】Funding 重复进入方向链

`funding_adjustment`（:7253）改 `tmv_final → tmv_blend`，被 `_tmv_vote` 读取；`_funding_vote`（:3141）又独立投一票。同一 Funding 状态两次影响方向。
**但量级有限**：调整有 0.20 上限+方向保护（不允许翻转符号，:7298-7303）；funding 票最大有效权重仅 0.20。低风险、可干净移除。

### 2.3 【小量级】GGR 正向 boost 与空间票

- 正 Gamma：`confidence_multiplier = 1.0+(1.15-1.0)·strength`，最大 ×1.15（:5838）——把"路径更稳"误转成"方向更确信"。
- pin 空间方向票（:5882），cap 0.25，**仅正 Gamma 生效**（负 Gamma `pin_trust=0`）。

负 Gamma 的 veto/cut 是正确的结构门，**保留**。正向 boost 与空间票量级小，去除主要是语义纯化。

### 2.4 【需消融证伪·高回归风险】SRD 0.70 权重方向票

`_srd_vote`（:3157）以 0.70×vote_confidence 投方向票，权重等同单个 CVD 窗、接近 TMV。Skew 的方向含义确实不如量价直接，但它是一个**大方向输入**。把它降为"仅严重反证"会抽走 0.70 权重，**可能在尾部行情降低方向命中**。必须靠消融实验 E 证伪，不能靠"skew 含义弱"的断言直接下线。

### 2.5 【中等】Macro 同时投票与硬阻断

`_macro_vote`（0.30，:3114）+ `MACRO_BLOCKING` veto（:2932）。普通宏观变化进方向票、极端值硬阻断；事件/时区/Macro 可能对同一美国风险重复处罚。拆分为事件门 + 背景上下文是合理的，量级中等。

### 2.6 【框定纠正】LLM"盲读"实为单次有序读，且泄漏 decision 与窗口

- `build_blind_theoretical_packet`（:129）**正确剔除** decision/signal_window/reasoning/blocking。
- 但 `build_prompt`（:178）在**同一次调用**里把盲读包与**完整包**一起喂给模型；完整包含 `decision`（:101）和 `signal_window`（:102）。schema 自陈 `derived_blind:False`、`single_call_blind_ordered`（:142-146）。

**关键纠正**：代码里的 `signal_window` 是 **DIE+Anchor 时序窗**，不是 v1.0 反复担心的"时区/session 耐久层"——后者在代码中**根本不存在**。因此真实问题是**"盲读其实不盲"（系统结论与窗口同次可见）**，而非"时区先验导致二次谨慎"。在时区层落地之前，不应以"时区二次谨慎"为由改 LLM 包。

### 2.7 【观测缺口】Anchor 修复机制未分类

存在 anchor_score / anchor_damage（:2420, :8150）但无 `repair_type`。三种修复路径（价格回锚 / 锚顺势迁移 / 混合）未拆开，无法判断 Alpha 来源。纯观测增强，零交易行为变化。

### 2.8 【已满足，移出问题清单】

- 单一总置信：**不存在**。无需"禁止"，只需在新 schema 中保持不引入。
- 多 confidence 语义：已是正交多字段 + UNCALIBRATED 标。风险是**人为合并**，靠展示层措辞约束，不是代码缺陷。
- 增强 GEX 双票：**代码已是"只降级"**，无需改动，仅需在文档明确这是要保持的不变量。

## 3. 封板设计原则（保留 v1.0，验证有效）

### 3.1 一个因子一个主角色

```text
TRIGGER / DIRECTION / CONFIRM / STRUCTURE_GATE /
SPATIAL_GATE / TEMPORAL_GATE / AUDIT_ONLY
```
除极端安全条件外，不允许一个因子同时拥有三种角色。

### 3.2 权限优先级

```text
硬安全门 > 窗口门 > 方向主干 > 确认器 > 修正项 > 审计参考
```

### 3.3 空间约束优先于弱方向票（卖方价差语境）

价格距 flip/wall/pin、负 Gamma 稳定性、可用空间，往往比弱 Macro/Skew 方向票更直接决定路径风险。

### 3.4 不构造单一总置信（已满足，保持）

最终输出正交维度，**禁止再引入** `final_total_confidence`：
```text
window_quality / direction_evidence_quality / structure_stability /
spatial_safety / temporal_durability / audit_dissent
```

### 3.5 所有角色调整先影子运行，且证不出改善就回退

新角色先产出 shadow package，不改旧 decision；至少完成一轮真实 replay 对比。**消融证不出"不劣于旧链"的项，保留旧链。**

## 6. 具体调整方案（按杠杆/风险重排优先级）

### 优先级 P1（高杠杆·低风险·先做）

#### 6.1 合并 CVD 4h/12h（→ FlowConfirmPackage）

```json
{
  "schema_name": "FlowConfirmPackage",
  "fast_4h": {}, "slow_12h": {},
  "agreement": "ALIGNED|CONFLICT|PARTIAL|MISSING",
  "absorption_state": "BUY_ABSORBED|SELL_ABSORBED|NONE",
  "combined_vote": 0.0,
  "combined_weight": 0.0,
  "data_quality": "OK"
}
```
合并原则：
- 两窗同向 → 一份确认（`combined_weight` 反映两窗一致度，**不是两票相加**）；
- 两窗冲突 → 降低 confirmation，保留 `absorption_state`，不形成两票互抵；
- `combined_weight` **封顶 ≤ TMV 有效权重**；
- EDB 只消费一份 `combined_vote`，且在 `_agreement` 中**只计一次**。

> 验证要点：合并后 CVD 在 score 与 agreement 中不再以 ≈1.47 主导；但"封顶 ≤TMV"是有方向后果的参数，须在消融 C 步确认不降低真实命中（若经验上流量更准，放宽封顶）。

#### 6.2 Funding 去重

二选一（不可叠加）：
- **首选**：新增 `edb_funding_direction_vote_enabled=False`，关闭 `_funding_vote`，保留 `funding_adjustment` 对 TMV 的拥挤修正；
- 备选：保留 `_funding_vote`，改 `funding_adjustment` 不动方向只出警示。

Funding 输出语义保持：`NEUTRAL / MILD_CONFIRM / CROWDED_WARNING / OPPOSITE_FUEL / HARD_WARNING`，其中 `MILD_CONFIRM` 不加方向、`HARD_WARNING` 硬阻断。

#### 6.3 LLM 真盲读修复（零交易行为变化）

- 首选：把盲读做成**真正独立的一次调用**（先盲读出 `theoretical_active_view`，再单独喂完整包做复核）；
- 若坚持单次调用：完整包中**移除 `decision` 的结论性字段**与 `signal_window`，并停止声称"盲读"，改称"有序复核"。
- LLM 始终 `AUDIT_ONLY`，不得改 direction/EDB/blocking/trade_allowed（现已满足，保持）。

### 优先级 P2（中风险·需消融）

#### 6.4 GGR 正向角色收口

- `ggr_positive_conf_boost_max` 由 1.15 改为 **1.0**（正 Gamma 不再抬方向置信）；
- 正 Gamma 输出 `structure_stability=SUPPORTIVE`（结构稳定性，不进方向分）；
- 新增 `ggr_spatial_vote_enabled=False`，pin/flip/walls 转入 `SpatialConstraintPackage`；
- **保留**负 Gamma veto/cut（已正确）。

#### 6.5 Macro 降级

拆 `MacroEventGate`（高影响事件/VOLQ shock → 硬门）与 `MacroBackgroundContext`（普通 DXY/US10Y → 审计背景或至多小幅 caution）；避免事件黑名单 + 时区 + Macro 三重处罚同一风险。

### 优先级 P3（高回归风险·最后做·必须先消融证伪）

#### 6.6 SRD 降级（高风险）

主链只用 `SEVERE_COUNTER_EVIDENCE / TAIL_HEDGE_DEMAND_HIGH / NEUTRAL / MISSING`；严重反向且数据充分时把 `APPROVABLE → REVIEW_REQUIRED`，不直接生成方向。
> **门槛**：必须在消融 E 步证明"去掉 SRD 0.70 方向票后，方向命中与尾部失效率不劣于旧链"，否则保留 SRD 方向角色。这是全方案回归风险最高的一步。

### 前向建设（依赖未落地部件）

#### 6.7 时区独立门（两步）

先按 `docs/信号时区置信度分类_理论与落地_v1.0.md` 落地时区层并校准，再接门：
```text

LOW + WEAK → WAIT_CONFIRMATION；EVENT_BLACKOUT → BLOCK/WAIT
```
时区门不应阻塞 P1–P3。

#### 6.8 Anchor 修复分类

## 9. 消融与验证计划（单变量、按风险升序）

```text
A  Legacy full EDB
B  A − Funding duplication        (P1, 预期低影响)
C  B + merged FLOW_CONFIRM        (P1, 关注 CVD 封顶对命中的影响)

F  E + Macro downgrade            (P2)
G  F + LLM true-blind             (无行为变化, 仅看 reason 可解释性)
```

主指标：`selection_rate / premise_failure_rate_60m / direction_accuracy / p95_NMAE / strong_grade_failure_rate / reason_code_coverage`。
不以回测 PnL 作为本轮唯一标准（本轮目标是信号结构封板）。

**护栏（本版新增）**：
- **选择率下限**：收口不得把 `selection_rate` 压到无法为 UNCALIBRATED 置信阶梯积累前向标签的程度（收口与校准争夺同一批样本）。设定一个最低选择率阈值，触发即放宽封顶/降级力度。
- **不劣回退**：任一消融步使方向命中或尾部失效率劣于旧链，则该步保留旧链角色（特别是 E 步 SRD）。

## 10. 结构模拟的定位（v1.0 §11 的诚实降级）

> v1.0 §11 的合成模拟（选择率 −9.5%、方向 +3.33pct、成功率 +4.32pct 等）是**机制自洽性检查，不是优化证据**。

该模拟**预设**了"CVD4 / CVD12 / Funding 共同部分观测一个独立于长期方向的瞬时 flow shock"。在此生成假设下，去重**必然**降低共同噪声的重复计权、提高方向正确率——这是设定的代数后果，对真实 BTC **无外部效力**，也不证明任何阈值适用真实市场。

它能说明的：去重在信息论上自洽（相关代理不应获得独立决策权，条件互信息 `I(Y;Z|X)=H(Y|X)−H(Y|X,Z)` 在 Z 为 X 的噪声代理时趋零）。
它不能说明的：在真实数据上净改善的**幅度**，乃至 CVD 封顶值、SRD 降级是否净正向——这些只能由 §8 Phase B/C 的真实 replay 与 §9 消融决定。

## 11. Research Agent 任务书

- **R0 因子血缘矩阵**：每个最终字段依赖哪些原始数据（price/volume/agg trades/funding/Deribit greeks/GEX API/macro），定位同源重复。

## 13. 封板验收标准

### 13.1 架构标准

- [ ] 每个因子有唯一 primary role；
- [ ] Funding 不重复计权；
- [ ] CVD 只形成一份 flow evidence 且 `combined_weight ≤ TMV`；
- [ ] GGR 正向不再 boost 方向、空间票退出 EDB（负向门保留）；
- [ ] Pin/Wall/Flip 进入空间包；
- [ ] LLM 盲读真盲（不含 decision/signal_window）；
- [ ] 不存在 total confidence（保持）；
- [ ] 所有状态均有 reason code；
- [ ] old/new 可 replay 对比。

### 13.2 验证标准（真实样本，非模拟）

- [ ] 至少一个历史样本外窗口；
- [ ] 新链选择率不异常膨胀，且不低于校准所需下限；
- [ ] 前提失效率或 p95 NMAE 不劣于旧链；
- [ ] 强信号失败率不升高；
- [ ] **SRD 降级在尾部 episode 不降低方向命中**（E 步硬门）；
- [ ] 人工审核可解释性提高；
- [ ] 无单一季度依赖；
- [ ] 未通过项明确保留 shadow，不强行上线。
