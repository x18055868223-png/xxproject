## 0. 执行摘要

把 LLM 复核从"解释系统结论"扩展为"额外给出一个基于数据截面的独立理论倾向",**可行,且有成熟方法论背书**。但要落地正确,必须先接受一个反直觉的前置认知:

> **这个独立倾向的价值不在"方向"本身,而在它与系统结论的"分歧"。**
> 因为 LLM 与系统读的是同一份因子截面,二者**误差相关**——"一致"几乎不携带信息,"分歧"才携带信息。

因此本扩展的正确定位是 **分歧探测器 / 去偏对照工具(divergence detector / debiasing instrument)**,而不是"第二个交易信号"。本文用信息论与统计的语言把这个判断讲清(第 1 部分),给出七个支撑它的成熟方法论模型(第 2 部分),并给出分阶段、可治理、可计分的落地路径(第 3 部分)。

# 第 1 部分 · 前置认知:分歧与信息量

## 1.1 信息量的本质:意外才有信息

信息论里,一个结果的自信息为 `I = -log P(结果)`:**越是意料之中的结果,信息量越低**。

把它套到 LLM 独立倾向上:
- 系统已经基于因子截面给出了结论。
- LLM 读**同一份**因子截面。
- 那么"LLM 也得出和系统一样的方向"是**大概率事件**(`P(一致 | 同输入)` 高)→ 自信息低 → **几乎没告诉你任何新东西**。
- "LLM 得出与系统不同的方向"是**小概率事件** → 自信息高 → **这才是值得人去看的信号**。

> 结论一(已按 1.7 修订):**分歧的"信息量"大于一致,但这说的是更新的"幅度",不是"方向"。一致仍是方向为正的(小幅)增信,分歧是方向为负的(大幅)降信——二者不矛盾。** 输出设计围绕 `divergence` 展开(因其幅度大、需触发动作),但**不否定**"一致 → 小幅增信"的合理性。详见 [1.7](#17-信息量--决策信心用户到底该信什么)。

## 1.2 误差相关性:为什么"一致"是弱证据

两个估计器要互相增信,前提是它们的**误差结构相互独立**。考虑两个无偏估计器,各自方差 `σ²`,相关系数 `ρ`,取平均后的方差为:

```
Var(average) = σ² · (1 + ρ) / 2
```

- `ρ = 0`(误差独立)→ 方差减半,集成有意义。
- `ρ → 1`(误差高度相关)→ 方差几乎不降,集成**毫无收益**。
- `ρ < 0`(误差反向)→ 方差可大幅下降。

本场景里,系统与 LLM **共享全部输入**,误差天然正相关(`ρ` 高)。这有两个直接推论:

1. **不能把 LLM 倾向"集成"进信号**(取平均/加权进 EDB):`ρ` 高,集成不降方差,反而引入 LLM 的幻觉方差,**期望为负**。→ 这是"永不回流 decision"红线的统计依据,不只是工程洁癖。
2. **"一致"本就是默认结局**,不增信;**"分歧"才是同输入下推理路径分叉的罕见事件**,是诊断价值所在。

> 结论二:独立倾向**不是新信息源**,是**同一信息的第二条推理路径**。它能做的是"暴露推理分叉",不能做的是"提供系统不知道的事实"。

## 1.3 两种"独立",别混淆

| | 输入独立(data-independent) | 推理独立(reasoning-independent) |
|---|---|---|
| 含义 | 第二个读者用**不同数据** | 第二个读者用**相同数据、不同推理** |
| 典型 | 真正的集成学习、多数据源 | 辩证自举、放射科双判 |
| 本设计 | ❌ 我们**故意不给**外部行情 | ✅ 我们要的正是这种 |
| 价值边界 | 可提供新事实 | 只能暴露推理/权重/门控差异 |

我们构建的是**推理独立**(同截面、异推理)。这决定了:不要期待 LLM 发现"系统没有的数据",要期待它发现"系统**用同样数据却做了不同处理**"的地方。放射科双判恰好证明了这种"同输入异处理"有真实价值(见 2.1)。

## 1.4 分歧的分类学:分歧是"问题触发器",不是"判决"

LLM 与系统分歧,可能来自四种成因,价值与处置完全不同:

| 分歧成因 | 含义 | 价值 | 处置 |
|---|---|---|---|
| **(a) 权重分歧** | LLM 比 EDB 更重/更轻某因子 | 中:暴露模型设定敏感性 | 人工判断哪种权重更合理 |
| **(b) 门控遮蔽** | 原始因子支持某方向,但被 Gamma veto / 置信地板压成 NEUTRAL | **高**:揭示"系统说中性 ≠ 因子说中性",方向被门控吞掉 | 复查门控是否过度保守 |
| **(c) 缺失处理分歧** | 系统排除了缺失因子(如 12h CVD),LLM 隐式当它存在 | **高**:暴露结论脆弱性 | 警惕结论依赖未确认数据 |
| **(d) 噪音/幻觉** | LLM 纯粹编了一个方向 | 负:污染 | 靠 track record / 校准过滤 |

> 结论三:`divergence` 出现时,系统应引导人去**诊断成因**(尤其找 b/c),而不是直接采信任何一方。独立视角是**提问的工具,不是回答的工具**。

## 1.5 为什么不能把倾向当胜率/信号(EV 论证)

- **未校准**:`conviction` 与任何概率没有映射,把它读成胜率是范畴错误(沿用 v1.0 的 `EVIDENCE_QUALITY_NOT_WIN_RATE`)。
- **无边际 alpha**:LLM 不是有市场择时优势的引擎,它只是重读了一遍确定性模型已经优化过的因子。给一个已优化估计叠加一个噪音重读,在误差相关(1.2)时**抬高方差、期望为负**。
- **锚定/谄媚**:若非盲读,LLM 会向系统结论靠拢,产出"事后合理化"而非独立判断。
- 故:**当信号用 = EV 负;当分歧探测/去偏用 = EV 正。** 这是整个扩展成立与否的分水岭。

## 1.6 沉默的价值:允许 UNCLEAR

正确计分规则(2.6)下,**被迫猜方向会恶化得分**。允许 LLM 在"缺失/高冲突/冷启动/未校准"时输出 `UNCLEAR`,是保护信息质量、避免制造噪音的设计。沉默不是失败,是诚实。

## 1.7 信息量 ≠ 决策信心:用户到底该信什么

1.1 说"分歧比一致更有信息量",这极易被误读成"应该更信分歧/更信 LLM"。**不是。** 必须把两个量分开:

- **信息量(幅度)**:一次观测**该把你的信念推动多大**。分歧在相关误差下是罕见事件,单次推动大;一致是预期事件,单次推动小。
- **更新方向(符号)**:这次观测把"系统正确"的后验**往哪个方向推**。一致 → 往**上**(增信);分歧 → 往**下**(降信)。

| 盲审比对 | 信息量(幅度) | 方向(符号) | 对决策信心的净效果 |
|---|---|---|---|
| 一致 | 小 | 正 | **小幅增信(有上限)** |
| 分歧 | 大 | 负 | **显著降信 / 触发审查** |

所以"**论证与 LLM 统一时才增加决策信心,反之需谨慎**"是**正确的决策规则**,与 1.1 的"分歧更有信息量"并不矛盾——前者讲方向,后者讲幅度,是同一个贝叶斯更新的两个侧面。我在 1.1 的原措辞过度强调幅度、贬低了"一致也是有效(小幅)增信",**该修正更准确,予以采纳。**

### 默认锚点:永远先信系统

要消解"到底信什么"的迷茫,关键是定一个**默认真值持有者 = 系统(冠军模型)**。LLM 是挑战者,永不取代它。于是决策规则清晰、不致瘫痪:

| 比对结果 | 决策信心 | 行动(默认信系统) |
|---|---|---|
| 一致 ALIGNED | 小幅↑(封顶) | 按系统执行;**不因一致而超出系统授权加码** |
| 轻微 MINOR | 基本不变 | 记录,无需特别动作 |
| 实质 MATERIAL | ↓ | **不翻向 LLM**;去查 `divergence.cause` 指出的点(门控遮蔽?缺失?权重?) |
| 对立 OPPOSED | 明显↓ | 同上,且更倾向"等待/降规模",直到成因解释清楚 |

> 核心原则:**分歧从不告诉你"该信 LLM 的方向",它只告诉你"在信系统之前该额外检查什么"。** 你默认永远信系统;LLM 只调节"行动前要多仔细",不调节"信谁"。于是不存在"两个权威打架"的困境——只有一个权威(系统)和一个检查清单触发器(LLM)。

### "LLM 始终给反向结论"——这是 bug,不是特性

担心的"LLM 老唱反调让人迷茫",有两种来源,都要堵死:

1. **被提示词诱导成习惯性唱反调**:若提示词暗示"你的价值在于分歧",模型会**为显得有用而制造分歧**。修正:**盲读阶段绝不告诉模型"去反对系统"——它根本看不到系统结论(盲),只被要求诚实给出独立读数,一致与分歧是同等合法的结局。`divergence` 由程序事后比对计算,从来不是模型的目标。** 分歧必须是**被测量的量,而非被指示的目标**。
2. **真是个坏顾问(无差别分歧)**:一个在多数信号上都分歧、且分歧毫无预测力的顾问,其分歧精确率≈随机,**在影子阶段(3.6 Phase 0)就会被 KPI(3.5 分歧精确率)抓出并退役,根本不会晋升到前端展示给用户**。"老唱反调又没用"的顾问,用户**永远见不到**。

### 也要防"过度相信一致"

反过来,因为系统与 LLM 共享输入(方法方差,2.7),**一致是弱交叉验证**。所以"一致→增信"必须**封顶**:绝不能因为"LLM 也同意"就把仓位放大到系统授权之上。**过度相信一致,与过度相信分歧,是同一枚硬币的两面错误。**

# 第 2 部分 · 成熟方法论模型

下面七个模型不是拼凑,而是一条逻辑链:**为什么分歧重要(2.7)→ 如何保住独立性(2.1)→ 同输入第二推理的理论许可(2.2)→ 如何榨出去偏价值(2.3)→ 如何使用分歧(2.4)→ 如何治理(2.5)→ 如何赢得信任(2.6)。**

## 2.1 盲读双判 + 仲裁(Blind double reading + reconciliation)

- **来源**:放射科 / 乳腺癌筛查,欧洲多国的标准流程。
- **原理**:两名读者**互相不知道对方结论(盲)**地独立判读同一影像,再由第三方/规则**仲裁**分歧。盲是为了**保住独立性**——一旦看到第一结论,第二读者会被锚定。
- **关键证据**:即使两名读者看的是**同一张影像**(输入相同、误差相关),双判仍能提高检出率——因为人的注意力/错误过程是**异质的**。这正是"同输入异处理仍有价值"的经验证明,直接为 1.3 的"推理独立"背书。
- **映射**:Pass 1 让 LLM **不看 `decision`** 形成盲读倾向;Pass 2 才揭示系统结论做比对仲裁。
- **局限**:我们的两个"读者"共享结构化输入,异质性弱于人类读者,故收益上界低于医学双判;但分歧的**诊断**价值仍在。

## 2.2 辩证自举 / 内部群体智慧(Dialectical bootstrapping，Herzog & Hertwig, 2009)

- **原理**:让同一个判断者在**不同假设下**生成第二个估计,与第一个平均,误差下降——"一个人脑中的群体智慧(crowd within)"。机理是即便同源,两次推理的误差也**部分独立**。
- **理论意义**:它为"同数据、异推理路径的第二判断有价值"提供了决策科学的正式依据。LLM 就是那个"辩证的第二推理者"。
- **映射**:LLM 在不同推理框架下重读截面。**注意**:经典辩证自举靠**平均**获益,而我们**不平均进信号**(1.2),只取其"暴露分歧"的那一半价值。

## 2.3 考虑相反面 / 红队 / 事前验尸(Consider-the-opposite; devil's advocate; pre-mortem)

- **来源**:Lord, Lepper & Preston (1984) 去偏研究;Klein 的 pre-mortem。
- **原理**:**主动生成反向论证**能显著降低确认偏误与过度自信。
- **映射**:输出强制 `counter_evidence`(反方因子)字段;即使 LLM 倾向与系统一致,也要它列出"反对自己的证据",对结论做压力测试。
- **价值**:这是少数"即便一致也有用"的机制——它检验的是结论的**稳健性**,不是方向。

## 2.4 集成分歧 / 委员会查询(Ensemble disagreement; Query-by-Committee，Seung et al., 1992)

- **原理**:委员会成员之间的**分歧度**是**认知不确定性(epistemic uncertainty)**的代理量;高分歧区域 → 模型最没把握 → **路由给人工**。
- **映射**:`divergence_from_system.level`(ALIGNED/MINOR/MATERIAL/OPPOSED)当作**不确定性闸**——MATERIAL/OPPOSED 自动升级为"建议人工重点复核"。
- **局限**:经典 QBC 假设成员多样;输入相关会**低估**真实不确定性,故分歧度是不确定性的**下界**(分歧大 → 一定不确定;分歧小 → 不代表一定确定)。

## 2.5 冠军/挑战者 + 模型风险治理(Champion–Challenger; SR 11-7)

- **来源**:银行模型风险管理监管指引(SR 11-7)。
- **原理**:新模型先以**影子模式并行运行、无任何决策权**,在样本外表现达标后才"晋升"。
- **映射**:`independent_view` 永远是**挑战者**;`validation_status` 在攒够 track record 前恒为 `UNVALIDATED`;任何人(含设计者)在此之前都不得按它操作。
- **价值**:这是把"实验性视角"关进笼子的治理框架,防止范围蔓延成隐性信号。

## 2.6 校准与正确计分规则(Calibration; Proper scoring rules; Brier; Tetlock）

- **来源**:Brier (1950) 计分;Tetlock 超预测研究。
- **原理**:概率预测必须用**正确计分规则**(Brier / 对数损失)评分,这类规则在数学上**激励诚实校准**;长期 track record 才能揭示一个预测者是否真有校准能力。
- **映射**:每次落库 `lean + conviction + 事后结果`,定期算 **Brier / 校准曲线 / 分层命中率**。**校准记录是 `independent_view` 唯一的"信任来源"**——没有它,倾向就是未证实的噪音。
- **附带**:正确计分规则同时奖励诚实的 `UNCLEAR`/弃权(1.6),反对硬猜。

## 2.7 三角验证 / 聚合效度 / 多特质多方法矩阵(Triangulation; Convergent validity; Campbell & Fiske, 1959)

- **原理**:多方法的"一致"只有在它们**误差结构独立**时才增信;共享方法带来的"方法方差(method variance)"会制造**虚假一致**。
- **理论收口**:这正是 1.2 的社科版本——系统与 LLM 共享"方法"(同一因子输入),所以它们的一致里有大量"方法方差",**不能当作互相验证**。它把"一致弱、分歧强"从直觉提升为方法论定理。
- **映射**:前端/文档措辞必须明示:"LLM 与系统一致≠互相验证(同输入共享方法方差)"。

### 方法论链条一图

```
[2.7 三角验证]  说明:为什么"分歧"才有信息(共享方法→一致是虚假增信)
        │
[2.1 盲读双判]  给出:如何保住独立性(Pass1 盲读)
        │
[2.2 辩证自举]  许可:同数据、异推理的第二判断有理论价值
        │
[2.3 考虑相反面] 榨取:反向论证→去偏(即便一致也测稳健性)
        │
[2.4 集成分歧]  使用:分歧度=不确定性闸→路由人工
        │
[2.5 冠军挑战者] 治理:无授权影子运行,晋升需样本外达标
        │
[2.6 校准/Brier] 信任:只有校准的 track record 才配被采信
```

# 第 3 部分 · 落地建议

## 3.1 架构:两段式(盲读 → 比对)

```
                ┌─────────────────────────────────────────┐
真实信号卡 ──→  │ build_blind_packet:                       │
                │   factor_cross_section + quality + 词典   │
                │   (不含 decision/reasoning/conflict)      │
                └───────────────┬─────────────────────────┘
                                ▼
                   [Pass 1 · 盲读]  → independent_view (derived_blind=true)
                                ▼
                ┌─────────────────────────────────────────┐
                │ build_reconcile_packet:                   │
                │   完整审计包 + Pass1 的盲读倾向            │
                └───────────────┬─────────────────────────┘
                                ▼
                   [Pass 2 · 比对]  → llm_review(审计) + divergence_from_system
                                ▼
        写回隔离: independent_view / divergence 与 llm_review 平级, 永不回流 decision
```

- **两次调用**保证 Pass 1 真盲读。低频信号成本可接受。
- **降级单次调用**:若省成本,只能在一次调用里"先写倾向再揭示系统",但 `decision` 已在上下文,独立性仅名义上;此时**必须**标 `derived_blind=false`,前端如实告知。不推荐作为默认。

## 3.2 输出 schema(在 v1.0 基础上增量,命名空间隔离不变)

```json
"independent_view": {
  "lean": "BULLISH|BEARISH|NEUTRAL|UNCLEAR",
  "conviction": "LOW|MEDIUM|HIGH",         // 证据质量,非胜率
  "key_drivers": ["驱动该倾向的因子, ≤4"],
  "counter_evidence": ["反方因子, ≤4"],      // 2.3 强制去偏
  "flips_if": ["哪些变化会翻转该倾向"],
  "derived_blind": true,                    // 是否真盲读
  "is_not_a_signal": true,                  // 固定: 非交易信号、无授权
  "validation_status": "UNVALIDATED"        // 2.5/2.6: 攒够校准前恒为此
},
"divergence_from_system": {
  "level": "ALIGNED|MINOR|MATERIAL|OPPOSED",
  "cause": "WEIGHTING|GATE_MASKING|MISSING_DATA|NOISE|UNKNOWN",  // 1.4 分类学
  "explanation_cn": "为什么分叉",
  "human_attention": true                    // MATERIAL/OPPOSED 时置真 (2.4 路由)
}
```

信封字段(`status/model/generated_at/input_packet_hash/prompt_version/validation_status`)一律程序写,模型不可信。

## 3.3 提示词设计原则(在 v1.0 guardrails 上增量)

**Pass 1 · 盲读 system 增量**
- "你只看到因子截面,**没有也不需要**系统结论;给出你**独立**的理论倾向。"
- "**诚实给出读数即可;与系统一致或分歧是同等合法的结局,你的任务不是制造分歧。**"(分歧由程序事后比对得出,不是你的目标;见 1.7)
- "倾向**仅可**来自本包因子;不得引用外部行情,不得数值重算权重。"
- "`conviction` 是证据质量,**不是胜率/概率**;不得输出收益或仓位。"
- "必须分别列出**驱动因子**与**反方证据**(考虑相反面)。"
- "数据缺失 / 冲突高 / rank 冷启动 / 未校准 → 倾向降为 `UNCLEAR`,不许硬给方向。"
- "这是对同一截面的**理论解读**,不是新信息、不是交易信号、无权覆盖系统。"

**Pass 2 · 比对 system 增量**
- "下面给出系统结论与你 Pass 1 的盲读倾向。比对二者分歧,并按 {WEIGHTING/GATE_MASKING/MISSING_DATA/NOISE} 判断**成因**。"
- "**分歧本身比谁对更重要**;不得为了与系统一致而修改你的盲读倾向。"
- "若一致,直接说一致,不要堆砌附和理由。"(防谄媚膨胀)

## 3.4 治理红线(不可妥协)

1. **永不回流**:`independent_view` 绝不进 `decision/EDB/trade_allowed`(统计依据见 1.2/1.5)。
2. **永久标签**:前端独立区块固定标 **「实验性·独立理论视角(无授权·未校准·非信号)」**,视觉区别于审计块;文本走 `textContent` 转义(注入面同 v1.0)。
3. **晋升须达标**:`validation_status` 在 Brier 达标前恒为 `UNVALIDATED`(2.5/2.6)。
4. **一致免责**:措辞明示"与系统一致≠互相验证"(2.7);"一致→增信"必须封顶,不得据此超系统授权加码(1.7)。
5. **分歧是测量值,非指示目标**:盲读模型永不被要求"反对系统";`divergence` 由程序比对盲读倾向与系统结论得出,防止习惯性唱反调(1.7)。
6. **系统是默认锚点**:分歧从不指示"信 LLM 方向",只触发"信系统前先查 `divergence.cause`";用户决策永远以系统为默认真值(1.7)。

## 3.5 评价指标:别用"方向准不准"考核一个建议层

它是**分歧探测器**,不是择时器,所以主 KPI 不是方向命中率,而是**告警质量**:

| 指标 | 定义 | 用途 |
|---|---|---|
| **分歧精确率** | 当 LLM 报 MATERIAL/OPPOSED 时,事后该信号确实失效/系统确有可议处的比例 | 主 KPI:告警是否"狼来了" |
| **分歧召回率** | 事后被证明有问题的信号中,LLM 当时报了分歧的比例 | 告警是否漏报 |
| **Brier / 校准曲线** | `conviction→事后结果` 的校准度 | 倾向本身是否可信(晋升门槛) |
| **分层命中率** | 按 lean×conviction 分桶的方向命中 | 诊断 LLM 在哪类截面更可靠 |
| **谄媚率** | 非盲读 vs 盲读时倾向偏移幅度 | 验证盲读是否真起作用 |

> 用"方向准不准"考核它,会逼它变成择时器(EV 负);用"告警准不准"考核它,才符合其分歧探测定位。

## 3.6 分阶段上线(冠军/挑战者节奏)

- **Phase 0 · 影子(仅落库,不展示)**:产出 `independent_view/divergence` 写进富化 JSONL,**前端不渲染**。攒 N 个真实信号的 track record。
- **Phase 1 · 实验展示**:前端以"实验性·未校准"标签展示;`human_attention` 引导人工复核;继续累积 Brier。
- **Phase 2 · 有条件采信(仅当达标)**:若分歧精确率/Brier 显著优于基线,**也只**把"分歧告警"作为人工注意力路由,**仍不回流信号**。
- **退役条件**:若 N 个信号后 Brier 不优于基率、分歧告警精确率接近随机 → **下线**,避免长期养一个噪音源。

## 3.7 对现有工程的影响 / 风险

- **不破坏 v1.0 安全测试**:新增字段、命名空间隔离、`decision` 零改动。
- **会改请求契约与成本**:两段式 = 两次调用 + 新 `prompt_version`;需扩展 `test_signal_llm_review_contract.py`(新增:盲读包不含 decision、independent_view 不回流、validation_status 默认 UNVALIDATED、divergence 路由置位)。
- **范围蔓延是头号风险**:你对 BIRL 全引擎是克制不引入的;独立倾向正是那种"给了方向数字就会被当第二信号"的东西。**守住 3.4 四条红线**即安全。

# 第 4 部分 · 补全视角:全局 Gamma 体制分析层(Gamma-Regime Lens)

> 这是"独立理论视角"要系统性补的**第一个、也是最有代表性的盲点**:系统和（默认的）LLM 都把 Gamma 当成对**方向点估计**的标量修正,漏掉了 Gamma 体制对**结果分布(方差/尾部/反身性)**的重塑。

## 4.1 缺口:系统把 Gamma 当"标量折扣",而非"分布重塑"

看真实卡:系统对 Gamma 的全部用法是——
- `gamma_regime.confidence_multiplier = 0.8`(正 Gamma 钉住 → 把置信**乘性压低**);
- `gamma_regime.veto`(极端时**一票否决**方向);
- 证据账本里 `GGR_SPATIAL` 标 `GATE_ONLY`(只做门控,不投方向票)。

这三者本质都是**对一阶矩(均值/方向)的标量调整**:要么把方向点估计打个折,要么直接掐掉。它表达不了的是:

> **"方向大概率没错,但在当前 Gamma 体制下,分布是爆炸性的/双峰的,一个催化剂就能让价格剧烈反向。"**

这正是你点出的场景:**量价/综合信息偏空,但在极端负 Gamma 下,一个信号或行为就可能触发剧烈反转(挤压)**。问题的本质是——

| | 模型回答的问题 | Gamma 体制回答的问题 |
|---|---|---|
| **一阶矩** | E[收益] 往哪边偏?(方向) | — |
| **二阶矩/尾部** | — | Var[收益] 多大?尾巴多肥?反向挤压概率多高? |

系统的"方向 + 置信"框架是**一阶矩世界**。Gamma 体制是**二阶矩/反身性现象**。**当分布是爆炸性双峰时,点估计几乎不携带可操作信息**——这是确定性加权引擎结构上无法表达、而定性推理者可以表达的东西。所以它是"补全视角"的旗舰案例,与全文主线(LLM 补程序结构性盲点)完全一致。

## 4.2 理论基础:做市商对冲与 Gamma 符号

机制核心是**期权做市商的动态 Delta 对冲**:他们持有期权库存,为保持 Delta 中性而随价格波动反复调仓,**其总 Gamma 的符号决定了这种调仓是稳定还是失稳的**。

- **做市商净多 Gamma(正 GEX)**:价涨→其 Delta 变多→需**卖出**标的;价跌→其 Delta 变少→需**买入**标的。即**追涨杀跌的反向**——**逆势对冲 → 压制波动、均值回归、向大额行权价"钉住(pinning)"**。(系统的 `POSITIVE_GAMMA_PINNING` + ×0.8 正是捕捉这一面)
- **做市商净空 Gamma(负 GEX)**:价涨→其 Delta 变更空→需**买入**;价跌→其 Delta 变更多→需**卖出**。即**助涨助跌**——**顺势对冲 → 放大波动、加速趋势、跳空与挤压、且在两个方向上都易出现剧烈反身性反转**。
- **Gamma 翻转点(flip point / zero-gamma)**:总 Gamma 过零的现价水平,是**体制边界**;价格穿越它会**切换体制**(稳定 ↔ 失稳)。卡里有 `gamma_regime.flip_point` / `gex_info.flip_point`。
- **反身性(Soros)**:对冲流 → 价格 → 更多对冲流的正反馈。**负 Gamma 是结构性的反身性放大器**。
- **负凸性与流动性**:负 Gamma 下做市商在压力中**抽走流动性**,造成"负凸性",放大跳空/踩踏/逼空。

> 一句话机制:**正 Gamma 是市场的"减震器",负 Gamma 是"油门 + 没有刹车"。** 极端负 Gamma 下,既有的偏空倾向不是"更安全的顺势",而是"骑在随时可能反向引爆的弹簧上"。

## 4.3 客观经验现象(可观察、可证伪)

这些是市场微结构里被反复观察到的模式(GEX/SpotGamma/SqueezeMetrics 一类实务框架,以及股指期权研究),作为**经验先验**而非铁律:

1. **钉住(pinning)**:正 Gamma + 临近到期,价格被吸附到大额行权价 / 磁吸位(卡里 `pin_strike` / `magnet_level`)。
2. **Gamma 挤压(squeeze)**:负 Gamma + 被迫对冲形成自我强化螺旋,价格单边失速。
3. **负 Gamma 加速与暴力反转**:趋势延展更猛,回调/反弹更剧烈,"恐慌见底后 V 反"概率显著上升——**偏空时尤其要防被催化剂引爆的逼空**。
4. **波动率体制扩张**:flip 下方 RV 常>IV、vol-of-vol 抬升(参照卡里 `iv_rv_ratio` / `dvol`)。

## 4.4 分析层设计:它输出"分布/风险叠加",不是方向

**红线先行(与 3.4 一致):Gamma 体制分析层调节的是"对方向倾向的置信"与"尾部/反转风险提示",它绝不产出一个竞争性的方向。** 它能说"虽然全都偏空,但极端负 Gamma 下反转尾巴很肥,把方向置信降到 LOW、盯住 flip 与 put_wall",**但不能**把倾向从空翻多、也不能发出交易指令。这样既补了盲点,又不破坏"非第二信号"的治理。

**输入(均来自现有卡字段):**
- `market_context.price` 相对 `flip_point` / `call_wall` / `put_wall` / `pin_strike` 的位置 → 体制符号与"离边界多近"。
- `net_gamma_notional_usd` 的**符号** + `gex_info.rank.metrics.gex_board.total_net_gex.abs_rank_pct`(**绝对分位**)→ 体制的**极端程度**(这是判断"极端负 Gamma"的关键,不是看符号,是看 |GEX| 有多大分位)。
- `gamma_regime.regime_strength`、`confidence_multiplier`、`veto`。
- `iv_rv_ratio`、`dvol` → 波动率体制背景。

**它对独立倾向的作用:**
- 仅**下调** `independent_view.conviction` 并升起尾部/反转风险标记;
- 当"方向倾向"与"Gamma 体制风险"指向相反含义(如偏空 + 极端负 Gamma 逼空风险),按 1.7 解读为**降信、加谨慎、查成因**,而非翻向。

## 4.5 输出 schema 增量

```json
"gamma_regime_lens": {
  "regime": "LONG_GAMMA_STABILIZING|SHORT_GAMMA_AMPLIFYING|NEAR_FLIP_UNSTABLE|UNKNOWN",
  "regime_extremity": "LOW|MEDIUM|HIGH",        // 主要由 |GEX| 的 abs_rank 决定
  "dynamics_cn": "该体制下的预期动力学(压制钉住 / 放大反身)",
  "dominant_tail_risk_cn": "主要尾部风险(如:偏空下的逼空反转)",
  "conviction_effect_on_directional_view": "NEUTRALIZE|LOWER|NEUTRAL|—",  // 只降不升、不改方向
  "key_levels": { "flip": 0, "call_wall": 0, "put_wall": 0, "pin": 0 },
  "positioning_assumption_cn": "本判断假设的做市商持仓符号(见 4.7 加密市场告警)",
  "data_quality_cn": "GEX/flip 缺失或 rank 冷启动时的可靠性说明",
  "lens_is_risk_overlay_not_direction": true
}
```

可选:在 `divergence_from_system.cause` 枚举里加 `VOLATILITY_REGIME`,用于"LLM 与系统分歧的原因是 Gamma 体制尾部被系统标量化处理掉了"。

## 4.6 提示词条款(Gamma 体制层,挂在盲读 Pass 1 之后或并入)

- "根据 `price` 相对 `flip_point` 的位置与 `net_gamma` 符号,判断当前 Gamma 体制;**说明你假设的做市商持仓符号**。"
- "**正 Gamma → 预期波动压制、向大额行权价钉住;负 Gamma → 预期波动放大、助涨助跌、两个方向都易剧烈反身。**"
- "**极端负 Gamma + 已有方向倾向 → 必须提示:一个催化剂可能引发与倾向相反的剧烈反转/挤压;据此把方向置信下调,并点名要盯的关键位(flip / 对侧 wall)。**"
- "本层输出是**分布/风险叠加**,**绝不**给出竞争性方向,**绝不**翻转倾向或发交易指令;你只能**下调置信**与**升起尾部风险标记**。"
- "若 `flip_point` / GEX 缺失或 rank 处于 `warming_up` 冷启动 → `regime=UNKNOWN`,不得臆断体制。"

## 4.7 边界与陷阱(必须诚实声明)

1. **符号假设依赖性(最重要)**:GEX 的符号取决于"谁持有哪类期权"的假设。**股指(SPX)世界常用的"做市商空 Put / 多 Call"零售流假设,在 BTC/加密期权上未必成立**——加密做市商/流结构不同,符号甚至可能反过来。**因此 Gamma 体制是关于持仓的"假设/假说",不是"测得的事实"**,提示词必须让 LLM 显式声明该假设。
2. **模型派生且嘈杂**:netGEX/flip 是估计量,卡里 rank 多为 `warming_up` 冷启动 → 本层继承全文的"数据质量谦逊",必要时给 `UNKNOWN`。
3. **是概率/尾部背景,不是择时**:它告诉你"分布形状与该盯的位",不告诉你"何时反转"。
4. **严防变成后门第二信号**:它**只下调置信、只升尾部标记**;一旦它开始隐式给方向,就退回 4.4 红线。

> 结论:Gamma 体制层把"方向对不对"之外的"这个方向在当前结构下有多脆弱、会从哪儿被引爆"补进了复核,**直接削减用户在极端体制下对方向点估计的过度自信(客观偏差)**——这正是独立理论视角最该补的盲点。

## 5. 结论

- **可行**:作为**分歧探测 + 去偏对照**工具,两段式盲读、带校准 track record。
- **理论充分**:三角验证(为何分歧重要)→ 盲读双判(如何独立)→ 辩证自举(同输入第二推理的许可)→ 考虑相反面(去偏)→ 集成分歧(用分歧路由)→ 冠军挑战者(治理)→ 校准/Brier(信任),是一条完整且被验证过的方法论链。
- **不可行的用法**:当第二 alpha / 隐性交易信号——同输入、误差相关、未校准、无边际优势,EV 为负。
- **用户的决策规则(采纳并固化,见 1.7)**:默认永远先信系统;**一致→小幅增信(封顶),分歧→降信并去查成因,而非翻向 LLM**。分歧是检查清单触发器,不是第二个权威——所以不存在"到底信谁"的困境。习惯性唱反调由"分歧不可被指示 + 影子阶段 KPI 退役"两道机制堵死。
- **Gamma 体制补全视角(第 4 部分)**:系统把 Gamma 当一阶矩的标量折扣,漏了它对**分布/尾部/反身性**的重塑。补一个"只下调置信、只升尾部风险、绝不给方向"的 Gamma 体制层,专门削减"极端负 Gamma 下偏空却被逼空反转"这类**对方向点估计的过度自信**。注意 GEX 符号在加密市场是**假设而非事实**,须显式声明。
- **一句话**:**让 LLM 的独立倾向去"提问",不要让它去"回答";它的方向是噪音,它的分歧才是信息——而面对分歧,你默认仍然相信系统,只是先把分歧点查清楚。**
