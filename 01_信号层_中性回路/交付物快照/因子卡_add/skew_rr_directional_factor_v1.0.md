# SRD 因子卡：期权偏斜/风险逆转 方向因子 v1.0

> 第一性因子卡。目的：让你彻底明白"为什么引入 Skew/RR、它到底减少哪个不确定性、改变哪个决策、什么时候会骗你"。
> 配套：本因子是 EDB（到期窗口方向合成层）的一张**方向证据票**，不单独下单、不选腿。

---

## 1. 因子名称

**SRD = Skew Risk-reversal Directional**
中文名：**期权偏斜方向因子（25Δ 风险逆转）**
版本：v1.0（设计稿，阈值待真实数据校准）

---

## 2. 第一性目标（为什么存在）

它只回答一个问题：

> **对于我即将卖出的那个 24–72h 到期，期权市场自己正在为哪个方向付钱，且这种付费意愿在往哪边迁移？**

价格趋势（TMV）、现货主动流（CVD）、跨资产宏观（MPF）都**看不到期权需求面**。而你做的是卖期权：期权市场对**同一张到期**的方向定价，是离你交易目标"路径最短"的方向信息——它和价格/现货流条件独立，是贝叶斯信息回路里**边际信息量最高**的一块拼图。

它**不回答**：价格一定涨跌、该不该开仓、卖哪个行权价、权利金够不够（那是执行层）。

---

## 3. 它减少哪个不确定性 / 改变哪个决策

- **减少的不确定性**：到期窗口方向倾向中，"由期权需求/对冲结构驱动"的那一部分。
- **改变的决策**：EDB 的方向票（偏多→倾向卖 Put Spread，偏空→倾向卖 Call Spread）。当 TMV 中性/不明时，SRD 与 CVD、Macro 一起**有资格形成弱方向**（这是你明确要的：TMV 不显著时允许其他要素造倾向）。
- 若它**无法改变** EDB 的方向或置信，则按 soul.md §2 自动降级为观察项——这是它的准入考核线。

---

## 4. 核心定义

25Δ 风险逆转：

```text
RR_25 = IV(25Δ Call) − IV(25Δ Put)     （单位：波动率点 / %）
```

- `RR_25 > 0`：看涨方向被付费（call 需求 > put 需求）
- `RR_25 < 0`：看跌/避险被付费（put 需求 > call 需求）

归一化偏斜（跨波动率环境可比，去量纲）：

```text
skew_norm = RR_25 / ATM_IV
```

偏斜动量（frontline 真正读方向的方式）：

```text
ΔRR = RR_25(now) − RR_25(lookback)
```

---

## 5. ⚠️ 最关键的正确性前提（你没经验，这条必须先记住）

> **BTC 的 25Δ 偏斜长期结构性为负**（看跌权常年带溢价、避险需求是常态）。

所以**绝对不能"RR 为负 = 看空"**——否则会系统性偏空（这正是你实盘信号里"清一色偏空"风险的同源陷阱）。

正确读法：用**相对自身近期基线的偏离**与**动量**，而不是原始符号。

```text
rr_z = (RR_25 − rolling_median(RR_25)) / rolling_scale(RR_25)   # 滚动分位/稳健z
方向信息 ≈ f(rr_z, ΔRR)，而不是 sign(RR_25)
```

- 偏斜从 −8% 修复到 −3%（仍为负，但在**变正**）→ **看涨**迁移。
- 偏斜从 −3% 急跌到 −9%（避险抢权）→ **看跌**迁移。

## 6. 数据源与口径

数据源：Deribit 公共接口（`https://docs.deribit.com/`），只读，无需私钥。

| 用途 | 端点 | 取用字段 |
| --- | --- | --- |
| 期号/行权价枚举（已实现） | `public/get_instruments` (kind=option) | `instrument_name / strike / option_type / expiration_timestamp / is_active / state` |
| 近月各档希腊字母与 IV | `public/ticker?instrument_name=...` | `greeks.delta`、`greeks.gamma`、`mark_iv`、`open_interest`、`underlying_price`、`index_price` |
| 指数现价（已实现） | `public/get_index_price` | `index_price` |

口径与取数策略（轻量、可控调用量）：

1. 对每个目标到期（最接近 +24h、+48h，最远不超过 72h）筛出活跃期权。
2. 只对**近月平值附近一小段**行权价调用 `ticker`（例如 ATM 上下各 ~6–10 档），拿到 `delta` 与 `mark_iv`。
   - 这一步**同时**为 GGR（全局 Gamma 因子）提供 `gamma × open_interest`，两个因子共用一次取数。
3. 在 call 侧找 `delta` 最接近 `+0.25`、put 侧最接近 `−0.25` 的两档，**在相邻档间对 IV 做线性插值**得到 `IV(25Δ)`，避免单档跳变。
4. `ATM_IV` 取 `delta` 最接近 ±0.5 的档。
5. 质量校验：两腿都需有效报价 / 满足最小 `open_interest` / 期号与目标到期一致；否则降权或放弃该到期。

## 7. 输出结构（进入 EDB 的方向票）

```json
{
  "factor_name": "SRD",
  "factor_version": "v1.0",
  "per_expiry": {
    "24h": {
      "expiry": "BTC-...",
      "hours_to_expiry": 23.8,
      "call_25d_iv": 0.512, "put_25d_iv": 0.560, "atm_iv": 0.535,
      "rr_25": -0.048, "skew_norm": -0.090,
      "rr_z": 0.83, "delta_rr": 0.011,
      "data_state": "OK"
    },
    "48h": { "...": "..." }
  },
  "vote": 0.34,
  "vote_confidence": 0.6,
  "lean": "BULLISH_TILT",
  "reason_codes": ["SKEW_REPAIRING_FROM_BASELINE"],
  "interpretation_cn": "偏斜仍为负但在向上修复、且短动量转正，期权需求面给出温和看涨迁移。"
}
```

- `vote ∈ [−1,+1]`：由 `rr_z` 与 `delta_rr` 合成的有符号方向票（**不是**原始 RR 符号）。
- `vote_confidence ∈ [0,1]`：数据质量 × 偏离/动量幅度；期号过近（<~6–12h）、报价稀薄、单腿缺失→降低。
- 多到期合成：向更贴合卖方腿的到期加权（48h 略重），冲突则降低 confidence。

## 8. 在 EDB 中的使用方式

- 作为**方向证据票**进入 EDB 加权合成（与 TMV、CVD×价格、Macro、Funding 并列）。
- TMV 显著时，SRD 主要起**确认/背离**作用（同向加稳、反向拉低一致度→更易判中性）。
- TMV 中性/不明时，SRD 允许**联合 CVD、Macro 造一个弱方向**。
- 与 CVD 的语义协同：SRD（期权需求面）与 CVD×价格（现货主动流）若同向→证据独立且共振，置信显著上升；若背离→真实冲突，EDB 应老实走中性（这正是"方向不明也是结论"的合法来源）。

---

## 9. 失败场景（什么时候它会骗你）

1. **结构性负偏斜被误读为看空** → 用相对基线+动量，已在 §5 设计规避。
2. **末日临近（<6–12h）**偏斜被 gamma/钉价噪声主导，非方向 → 临近到期降权。
3. **事件性避险抢权**（put 抢权是买保险，不是方向信仰）→ 用动量+与 CVD/TMV 交叉确认，不单独放大。
4. **翼部流动性稀薄** → 25Δ IV 不可靠 → 最小 OI/有效报价门槛，否则降权或跳过。
5. **插值跨越稀疏行权价** → 偏差大 → 记录所用两档，质量低则降权。

## 10. 参数表（**全部待真实数据校准，非理论最优**）

| `srd_delta_rr_lookback` | 待定（如 数小时） | 偏斜动量回看 |
| `srd_min_oi` | 待定 | 单腿最小未平仓量 |

校准前不得把初值当成已验证 alpha；校准必须版本化（soul.md §6/§11）。

## 11. 记录与验证

- 每轮记录：各到期 `rr_25 / skew_norm / rr_z / delta_rr / 所用两档strike与IV / vote / vote_confidence`。
- 红灯断言（落地时）：
  - 构造已知 IV 的合成链 → RR 符号正确；
  - **结构性为负但在修复**的偏斜 → vote 看涨（不得看空）；
  - 单腿缺失/稀薄 → 降权或 data_state≠OK，不污染 EDB；
  - 不得返回任何选腿/报价/下单字段（守 v0.4 执行边界）。

---

## 12. 一句话总结

**SRD 只做一件事：用同一张 24–72h 到期的 25Δ 风险逆转的"相对偏离与动量"，把期权市场自己的方向定价，作为一张独立方向票喂进 EDB——而不是把常年为负的偏斜误读成永远看空。**
