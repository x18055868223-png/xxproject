# 宏观压力因子文档 v1.0

## 1. 因子名称

**MPF = Macro Pressure Factor**  
中文名：**宏观压力因子**

MPF 是一个基于公开宏观市场代理变量的环境压力因子，用于判断最近约三日窗口内，外部宏观环境对风险资产更偏向逆风、顺风还是中性。

MPF 当前由三个组件构成：

```text
VOLQ  ：波动率压力代理
DXY   ：美元强弱代理
US10Y ：美国10年期收益率代理
```

三个组件经过统一的变化率计算、强度分层、方向保留和权重合成后，输出一个标准化的宏观压力分数。

---

## 2. 因子目标

MPF 的目标不是预测目标资产价格，也不是直接生成交易信号，而是回答一个更基础的问题：

> 最近约三日内，宏观市场变量是否共同形成了可识别的风险资产逆风或顺风？

它回答：

1. 波动率压力是否明显上升或下降；
2. 美元强弱是否形成流动性压力或缓和；
3. 长端利率是否形成折现压力或缓和；
4. 三个组件合成后，宏观环境是逆风、顺风还是中性；
5. 当前结论的数据可用性和可信度如何。

它不回答：

1. 目标资产是否一定上涨或下跌；
2. 当前是否适合建仓、平仓或调仓；
3. 趋势是否延续；
4. 价格结构、成交量、仓位或资金费率是否确认；
5. 任何下游策略是否应该否决或放大信号。

---

## 3. 因子定位

MPF 是一个**外生宏观环境因子**，不是价格趋势因子，也不是风险控制规则。

它的输出应该被理解为：

```text
macro_score > 0 ：宏观逆风，数值越大压力越强
macro_score < 0 ：宏观顺风，数值越小支持越强
macro_score = 0 ：宏观信息不足或接近中性
```

MPF 只描述外部环境，不改写其他因子的方向、强度或事件判断。下游系统可以读取它作为补充上下文，但不应把它当作单独的买卖条件。

## 4. 数据源与代理变量

### 4.1 代理变量表

| 组件 | 主代码 | 备用代码 | 含义 | 正向变化含义 |
| --- | --- | --- | --- | --- |
| `VOLQ` | `^VOLQ` | `^VXN`, `^VIX` | 纳指相关波动率压力代理 | 波动率压力上升 |
| `DXY` | `DX-Y.NYB` | `DX=F` | 美元指数代理 | 美元走强，流动性压力上升 |
| `US10Y` | `^TNX` | 无 | 美国10年期收益率代理 | 长端利率上升，折现压力上升 |

建议配置：

```python
MACRO_SYMBOLS = {
    "VOLQ": "^VOLQ",
    "DXY": "DX-Y.NYB",
    "US10Y": "^TNX",
}

MACRO_SYMBOL_CANDIDATES = {
    "VOLQ": ["^VOLQ", "^VXN", "^VIX"],
    "DXY": ["DX-Y.NYB", "DX=F"],
    "US10Y": ["^TNX"],
}
```

### 4.2 备用代码语义

备用代码只用于提高数据可用性，不表示完全等价：

1. `^VXN` 比 `^VIX` 更接近 `VOLQ` 的科技股波动率语义；
2. `^VIX` 是更宽泛的股票波动率代理，使用时应在输出中保留实际 `source_symbol`；
3. `DX=F` 是 DXY 的期货代理，和现货指数可能有轻微差异；
4. `US10Y` 暂不建议用其他期限利率替代，避免改变因子含义。

## 6. 数据清洗与可用性

实现时必须满足：

1. 按 `timestamp` 升序排列；
2. 重复 `timestamp` 只保留最新一条；
3. `close <= 0` 的记录视为无效；
4. 空数据、HTTP 错误、超时和解析错误不得向外抛出；
5. 至少需要当前有效收盘价和参考收盘价，才能计算该组件；
6. 对传统金融市场休市不做本地日历硬编码，只根据实际可用数据判断；
7. 每个组件必须输出 `source_status`，至少包含 `live`、`lkgv_cache`、`unavailable`。

---

## 7. LKGV 缓存

由于三个宏观代理变量依赖传统金融交易日，周末、节假日或接口异常时可能取不到实时有效数据。MPF 使用 LKGV 缓存提高稳定性。

```text
LKGV = last known good value，最后已知有效值。
```

建议缓存文件名：

```python
MACRO_CACHE_FILE = "macro_factor_cache.json"
```

处理规则：

1. 先请求主代码；
2. 主代码失败时尝试备用代码；
3. `VOLQ` 获取不到可用历史 close 时，依次降级到 `^VXN`、`^VIX`；
4. 所有候选都失败时，读取本地 LKGV 缓存；
5. 缓存可用时，`source_status = "lkgv_cache"`；
6. 缓存也不可用时，`source_status = "unavailable"`；
7. 不因宏观数据失败中断主流程，只降低因子可信度。

建议缓存记录包含：

```json
{
  "component": "VOLQ",
  "source_symbol": "^VOLQ",
  "timestamp": 1716508800,
  "close": 20.5,
  "bars": [],
  "updated_at": 1716530000
}
```

## 8. 三日 bps 净变化

MPF 使用最近有效日线与约 72 小时前的有效日线比较，计算三日净变化。

参考点选择：

```text
current = 最近一个有效日线 close
reference = timestamp <= current_timestamp - 72h 的最近一个有效日线 close
```

如果严格 72 小时前没有记录，则使用更早的最近有效记录。这样可以兼容周末和节假日。

### 8.1 计算公式

`VOLQ` 和 `DXY` 使用相对变化：

```text
bps = (current_close / reference_close - 1) * 10000
```

`US10Y` 使用收益率绝对变化：

```text
bps = (current_close - reference_close) * 10
```

原因：

```text
Yahoo 的 ^TNX 报价约等于 10年期收益率 * 10。
因此 ^TNX 变化 1 个点，约等于收益率变化 10 bps。
```

### 8.2 方向含义

三个组件统一采用以下方向：

```text
bps > 0 ：宏观压力上升，对风险资产偏逆风
bps < 0 ：宏观压力下降，对风险资产偏顺风
```

强度计算使用 `abs(bps)`，方向符号在合成分数时保留。

## 9. 强度分层

每个组件使用独立阈值，因为三个变量的自然波动量级不同。

```python
MACRO_BPS_TIERS = {
    "VOLQ": [150, 300, 450, 700],
    "DXY": [20, 45, 80, 130],
    "US10Y": [4, 9, 16, 28],
}
```

分层结果：

| 绝对 bps 区间 | strength | tier | 中文 |
| --- | ---: | --- | --- |
| 小于第一档 | `0.00` | `noise` | 噪音 |
| 第一档到第二档 | `0.22` | `mild` | 轻微 |
| 第二档到第三档 | `0.48` | `moderate` | 中等 |
| 第三档到第四档 | `0.74` | `strong` | 强 |
| 大于第四档 | `1.00` | `extreme` | 极端 |

阈值换算：

| 组件 | mild | moderate | strong | extreme | 解释 |
| --- | ---: | ---: | ---: | ---: | --- |
| `VOLQ` | 1.50% | 3.00% | 4.50% | 7.00% | 波动率指数三日相对变化 |
| `DXY` | 0.20% | 0.45% | 0.80% | 1.30% | 美元指数三日相对变化 |
| `US10Y` | 4 bps | 9 bps | 16 bps | 28 bps | 10年期收益率三日绝对变化 |

## 10. 合成分数

### 10.1 组件权重

```python
MACRO_COMPONENT_WEIGHTS = {
    "VOLQ": 0.42,
    "DXY": 0.33,
    "US10Y": 0.25,
}
```

权重含义：

1. `VOLQ` 权重最高，因为波动率压力通常最直接反映风险偏好变化；
2. `DXY` 代表美元流动性环境，是风险资产的重要外部压力；
3. `US10Y` 代表折现率压力，但三日窗口内相对更慢，因此权重略低。

### 10.2 组件分数

```text
component_score = sign(bps) * strength * weight
```

示例：

```text
VOLQ 三日上涨 3.4%
abs_bps = 340
tier = moderate
strength = 0.48
weight = 0.42
component_score = +0.2016
```

### 10.3 数据可信度

```text
all live: 1.00
含 lkgv_cache 且无 unavailable: 0.72
含 unavailable: 0.65
全部 unavailable: 0.00
```

建议不对缺失组件重分配权重。缺失组件的 `component_score` 记为 0，并通过 `macro_data_confidence` 降低总分可信度。这样可以避免两个可用组件代替缺失组件发出过强结论。

### 10.4 总分

```text
raw_score = sum(component_score)
macro_score = raw_score * macro_data_confidence
```

范围：

```text
macro_score ∈ [-1, +1]
```

## 11. 状态分类

MPF 建议把方向状态、特殊压力标记和数据状态分开输出。

### 11.1 方向状态

| macro_regime | 触发条件 | 含义 |
| --- | --- | --- |
| `Headwind` | `macro_score >= 0.46` | 明确宏观逆风 |
| `Mild Headwind` | `0.18 <= macro_score < 0.46` | 温和宏观逆风 |
| `Neutral` | `-0.18 < macro_score < 0.18` | 宏观信息不足或相互抵消 |
| `Mild Tailwind` | `-0.46 < macro_score <= -0.18` | 温和宏观顺风 |
| `Tailwind` | `macro_score <= -0.46` | 明确宏观顺风 |

### 11.2 特殊压力标记

| flag | 触发条件 | 含义 |
| --- | --- | --- |
| `VOLATILITY_RISING` | `VOLQ bps > 0` 且 `VOLQ tier >= moderate` | 波动率压力明显上升 |
| `VOLATILITY_SHOCK` | `VOLQ bps >= 450` | 波动率冲击 |
| `MIXED_MACRO` | 正负组件同时达到 `moderate` | 宏观组件方向冲突 |

建议参数：

```python
MACRO_VOLQ_SHOCK_BPS = 450
```

### 11.3 数据状态

| data_status | 触发条件 | 含义 |
| --- | --- | --- |
| `full_live` | 三个组件均为 live | 数据完整且新鲜 |
| `cached` | 至少一个组件来自 LKGV，且无 unavailable | 使用最后有效值 |
| `partial` | 至少一个组件 unavailable | 数据不完整 |
| `unavailable` | 三个组件均 unavailable | 不输出有效宏观判断 |

## 12. 输出结构

建议输出完整结构，而不是只输出一个分数。

后续项目基于本文档生成因子时，必须把 `VOLQ / DXY / US10Y` 的当前实际数据和三日实际变化作为判断依据一并展示，不能只显示 `MPF`、`macro_score` 或 `macro_regime`。MPF 的结论必须能够被用户直接追溯到三个组件的原始读数。

每个组件至少展示：

```text
component          组件名称
source_symbol     实际使用的数据代码
source_status     live / lkgv_cache / unavailable
current_close     最近有效收盘值
reference_close   三日前参考收盘值
change_pct_3d     三日实际变化百分比
scoring_bps       用于强度分层和合成评分的 bps
tier              分层结果
component_score   组件贡献分
```

其中 `change_pct_3d` 对三个组件都按实际报价的相对变化展示：

```text
change_pct_3d = (current_close / reference_close - 1) * 100
```

`scoring_bps` 则遵守第 8 节的评分口径：`VOLQ / DXY` 使用相对变化 bps，`US10Y` 使用收益率绝对变化 bps。这样既能展示用户可读的实际涨跌百分比，也能保留利率组件更合理的评分方式。

## 13. 实现伪代码

```python
def macro_strength(symbol_key, bps):
    tiers = MACRO_BPS_TIERS[symbol_key]
    x = abs(bps)

    if x < tiers[0]:
        return 0.0, "noise"
    if x < tiers[1]:
        return 0.22, "mild"
    if x < tiers[2]:
        return 0.48, "moderate"
    if x < tiers[3]:
        return 0.74, "strong"
    return 1.0, "extreme"


def component_score(symbol_key, bps, source_status):
    if source_status == "unavailable" or bps is None:
        return {
            "strength": 0.0,
            "tier": "unavailable",
            "component_score": 0.0,
        }

    strength, tier = macro_strength(symbol_key, bps)
    weight = MACRO_COMPONENT_WEIGHTS[symbol_key]
    sign = 1 if bps > 0 else -1 if bps < 0 else 0

    return {
        "strength": strength,
        "tier": tier,
        "component_score": sign * strength * weight,
    }


def data_confidence(components):
    statuses = [c["source_status"] for c in components]

    if all(s == "unavailable" for s in statuses):
        return 0.0
    if any(s == "unavailable" for s in statuses):
        return 0.65
    if any(s == "lkgv_cache" for s in statuses):
        return 0.72
    return 1.0


def classify_macro(components):
    confidence = data_confidence(components)
    raw_score = sum(c["component_score"] for c in components)
    macro_score = raw_score * confidence

    if macro_score >= 0.46:
        regime = "Headwind"
    elif macro_score >= 0.18:
        regime = "Mild Headwind"
    elif macro_score <= -0.46:
        regime = "Tailwind"
    elif macro_score <= -0.18:
        regime = "Mild Tailwind"
    else:
        regime = "Neutral"

    return macro_score, regime, confidence
```

## 14. 阈值与逻辑合理性评估

### 14.1 三日窗口

三日窗口适合作为宏观压力的短周期观察口径：它比单日变化更抗噪，又不会像 20 日窗口那样反应迟缓。

限制是：传统市场有休市，三日窗口在周末和节假日前后实际可能跨越更长自然时间。因此实现必须保留参考点时间戳，并在输出中暴露数据来源。

### 14.2 VOLQ 阈值

`VOLQ` 阈值 `[150, 300, 450, 700]` 对应三日相对变化 `1.5% / 3.0% / 4.5% / 7.0%`。

这个设置适合作为早期风险提示：波动率指数本身弹性较高，4.5% 不一定是历史极端，但足以说明波动压力已经有明显变化。若用于严格的冲击识别，建议后续用历史分位数校准，例如使用过去 3 年三日变化的 60% / 75% / 90% / 97% 分位作为四档阈值。

### 14.3 DXY 阈值

`DXY` 阈值 `[20, 45, 80, 130]` 对应三日相对变化 `0.20% / 0.45% / 0.80% / 1.30%`。

这个量级整体合理。DXY 的日常波动远低于波动率指数，因此 0.8% 的三日变化已经能代表较明显的美元压力，1.3% 可视为强美元冲击。

### 14.4 US10Y 阈值

`US10Y` 阈值 `[4, 9, 16, 28]` 对应三日收益率绝对变化 `4 / 9 / 16 / 28 bps`。

这个量级适合作为利率压力识别。4 bps 以下多半属于噪音或常规波动，16 bps 以上通常已经值得标记为显著利率变化，28 bps 以上可以视为强冲击。

### 14.5 权重

`VOLQ 0.42 / DXY 0.33 / US10Y 0.25` 的分配是合理的初始版本：

1. 波动率压力最直接，权重最高；
2. 美元指数对全球流动性敏感，权重居中偏高；
3. 长端利率重要，但三日窗口内传导相对更慢，权重略低。

如果目标资产对利率更敏感，可以提高 `US10Y` 权重；如果目标资产更受美元流动性驱动，可以提高 `DXY` 权重。权重调整后必须保持总和为 1。

### 14.6 regime 阈值

`0.18 / 0.46` 的方向状态阈值偏保守。单个轻微信号通常不会触发 `Mild Headwind` 或 `Mild Tailwind`，需要至少一个中等强度组件，或多个轻微组件同向共振。

这种设计有利于减少噪音，但也意味着因子对早期单组件变化不会过度反应。为了避免信息被合成分数抵消，输出中必须保留每个组件的独立状态和 `flags`。

### 14.7 数据可信度

`LKGV = 0.72`、`unavailable = 0.65` 的置信度折扣逻辑是合理的保守处理，但建议增加缓存年龄衰减：

```text
cache_age <= 72h      ：保留 0.72
72h < cache_age <= 7d ：降至 0.50
cache_age > 7d        ：视为 unavailable
```

这样可以避免长期陈旧缓存仍然影响宏观判断。

### 14.8 主要局限

MPF 是美国市场中心的宏观代理因子，适合观察美元、利率和波动率环境，但不覆盖所有宏观变量。它不包含：

1. 信用利差；
2. 流动性拆借利率；
3. 实际利率；
4. 商品价格；
5. 国家或地区差异；
6. 目标资产自身的价格、成交和仓位状态。

因此，MPF 只能作为宏观环境描述，不能替代目标资产自身的结构因子。

## 15. 维护规则

后续修改 MPF 时，必须遵守：

1. 不引入具体业务系统、策略名称或下游决策语义；
2. 不让宏观因子直接改写其他因子的输出；
3. 不把 `cached`、`partial`、`unavailable` 混入方向状态，应作为数据状态单独输出；
4. 不用本地时间硬贴“周末/节假日”等事件标签；
5. 备用代码必须保留 `source_symbol`，避免代理变量被误读为主变量；
6. 缺失组件不重分配权重，只通过可信度折扣处理；
7. 阈值调整必须同时说明对应的市场含义和历史校准依据。
8. 输出中必须展示 `VOLQ / DXY / US10Y` 的当前值、参考值、三日实际变化百分比和评分 bps，不能只展示合成后的 MPF 结论。

---

## 16. 最终定义

**MPF，Macro Pressure Factor，宏观压力因子，是一个由 `VOLQ / DXY / US10Y` 三个公开市场代理变量构成的外生环境因子。它通过三日 bps 净变化、分组件强度分层、固定权重合成和数据可信度折扣，输出宏观逆风、顺风或中性的标准化判断。MPF 只描述宏观环境，不生成交易建议，不改写其他因子的方向或强度。**

---

## 17. 一句话总结

**MPF v1.0 只做一件事：判断最近约三日内，波动率、美元和长端利率是否共同形成了可识别的风险资产宏观逆风或顺风。**
