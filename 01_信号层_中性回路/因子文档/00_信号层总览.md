## 0. 模块定位

中性回路是整合系统的 **① 信号层 = 前置信号 + 策略制定层**。

- 运行边界：`read_only_demo=True`，**不选腿、不报价、不下单**；`order_layer="external_execution_program"`（契约校验 `demo/contracts.py:139-141` 强制）。
- 它只回答两件事：**窗口何时开（时序）** 与 **到期窗口哪一侧收敛（方向）**；具体腿/数量/报价全部外置给执行层（②）。
- 整合中的角色：信号层是 `SignalEvidencePackage` 的**唯一生产者**（总纲 v0.3 §6.1）。下游执行层只消费这个包，不读 EDB 内部状态。

## 1. 架构：时序与方向解耦

这是 v0.5 EDB 重构的核心思想（`demo/edb.py:1-22`、`demo/FORWARD_DESIGN_V0.5_EDB.md`）。三层互不混淆：

```
                          ┌─ 主链路（进 MODULE_SEQUENCE / module_results）
公共行情 → External Gate → Anchor → TMV-F
                          │
                          ├─ 时序门（不进 module_results）
                          │   M-DIE ─┐
                          │          ├→ NeutralRepair 状态机 → is_active（窗口是否打开）
                          │   Anchor ─┘
                          │
                          └─ 方向层（权威，不进 module_results）
                              EDB 合成六类证据：
                                TMV · CVD_4h · CVD_12h · MACRO · FUNDING · SRD · GGR_SPATIAL
                              + 安全门：GGR 负Gamma veto/cut · MACRO 硬阻断 · FUNDING 极端拥挤 veto
                              → strategy_recommendation（signal / strategy_code / 期号 / edb_*）
```

**关键概念：DIE+Anchor 的 UP/DOWN 是噪声、不是交易方向**。窗口"何时开"由时序门定，"往哪一侧"由 EDB 独立定。EDB 的 `precondition.nr_active` 读 NeutralRepair 的 `is_active`：窗口未开时方向仅作观察预热（`NO_TRADE_BLOCKED`），不输出可交易方向。

## 3. 整合状态（关键：信号层零代码改动）

总纲 v0.3 §3.1 结论：**整合调整 = 零**。信号层从不依赖 KPF，EDB 六证据里没有任何 KPF 项。整合不改信号层代码，只通过**消费其输出**实现。

- **桥接（Phase 2 待建）**：从 `demo/logs/snapshots.jsonl` 或运行时快照导出 `SignalEvidencePackage`（schema `nrd.integration.signal.v0.1`，总纲 v0.3 §6.1）。映射：
  - `support_label`（EDB `support_label`）→ 执行层 `SIGNAL_STATE`
  - `direction_bias`（EDB `edb_score` 符号 / `side_hint`）→ 执行层 `DIRECTION_BIAS`（看空=SHORT_CALL=put credit? 见下方注）
  - `confidence`（EDB `confidence`）→ 执行层 `SIGNAL_CONFIDENCE`
  - `edb{edb_score,lean,agreement,coverage,ggr_gate}` → 对冲模块持续性输入
  - `NO_TRADE_*` / `WAIT_CONFIRMATION` → 不进执行计划
  - 注：EDB 内部 `side_hint`：`edb_score>0`（看多）→ `SIDE_PUT_CREDIT_SPREAD`；`edb_score<0`（看空）→ `SIDE_CALL_CREDIT_SPREAD`（`demo/edb.py:400-401`）。
- **文档卫生（repo 级，非因子级）**：`中性回路 - opus4.8\INTEGRATION_HANDOFF.md` 的"KPF 空间层三层叙事"、`PROJECT_MEMORY.md §16` 的"与下游 KPF 整合"已被总纲取代/作废。
