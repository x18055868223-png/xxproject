# 审计部署模块 · 组件总览

> 当前模块口径（r2.2 / 2026-06-19）：本模块是信号审计页面的策略外部署层。它读取 FMZ 已写出的 JSONL 与 LLM sidecar，生成静态审计卡；不回写 FMZ，不改变信号结论，不触发交易。

## 0. 模块定位

审计部署模块负责把信号层产物变成可浏览、可追溯、可复核的静态页面：

1. FMZ 信号层写入 `signal_review.jsonl`。
2. LLM sidecar 可选读取真实信号卡并写入 `signal_llm_reviews.jsonl`。
3. materializer 把 JSONL 和 sidecar 合并成 `signal_cards/`。
4. 静态前端读取 `signal_cards/index.json` 和单卡 JSON 展示。
5. Apache/Nginx 对外提供 `/signal-audit/`。

## 1. 组件清单

| # | 组件 | canonical | 一句话 |
| --- | --- | --- | --- |
| 01 | materializer | `tools/materialize_signal_cards.py` | JSONL -> `signal_cards/` |
| 02 | static frontend | `deploy/signal_audit/frontend/` | 静态审计卡阅读器 |
| 03 | LLM sidecar | `tools/gemini_signal_llm_review.py` + `run_signal_llm_review.sh` | 策略外 Gemini 复核 |
| 04 | systemd / paths | `signal-audit-*.service`、`*.timer`、`install_or_update.sh` | 服务器安装、定时刷新和路径约定 |

## 3. 边界

- 本模块是审计/展示/复核层，不是交易执行层。
- LLM 复核不改变 `decision`、`reasoning`、`blocking`、`trade_allowed`。
- committed frontend `signal_cards/` 是 fixture；生产数据由服务器 materializer 生成。
- `install_or_update.sh` 是主动部署更新，应在维护窗口运行。
