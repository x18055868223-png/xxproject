# 审计卡片语义文档

> 当前模块口径（r2.2 / 2026-06-19）：本文解释审计前端如何展示 `signal_review_card@1.0.0`、GEX rank 和 `llm_review`。它是展示语义文档，不改变 JSON 原始字段。

## 1. 输入文件

| 文件 | 语义 |
| --- | --- |
| `signal_cards/index.json` | 卡片 manifest，供列表和过滤使用 |
| `signal_cards/*.json` | 单张审计卡 |
| `signal_cards/fallback.js` | 静态 fallback fixture |
| `signal_llm_reviews.jsonl` | 服务器 sidecar，由 materializer 合并进单卡 |

## 2. 主要页面区块

| 区块 | 来源 | 展示边界 |
| --- | --- | --- |
| 顶部摘要 | `identity`、`decision`、`market_context` | 不把 confidence 当胜率 |
| Gamma/GEX 重点 | `factor_cross_section.gamma_regime`、`gex_info` | 外部 GEX 与内部 GGR 分源展示 |
| GEX Rank 分位 | `factor_cross_section.gex_info.rank` | 展示 `quality=warming_up`，区分 netGEX 方向分位和绝对值分位 |
| 决策结论 | `decision`、`reasoning` | 系统结论为主，LLM 不覆盖 |
| 数据质量与时效 | `quality.sources` | 缺失字段必须可见 |
| LLM 复核 | `llm_review` | 独立审计建议层，不改系统信号 |

## 3. 中文语义优化规则

- 展示层可以翻译 label，但不得改写原始 enum 和数值。
- 同质化内容要合并：如果 LLM 已列出关键价位或证据，上方摘要不重复展开长段解释。
- “暂缺”需要尽量附带字段名、来源和缺失原因。
- `rank_pct`、`confidence`、`agreement` 都不是收益预测或胜率。

## 4. 缺失兼容

页面必须兼容：

- 没有 `llm_review` 的旧卡。
- 没有 GEX rank 的旧卡。
- optional GEX 数据缺失但 required sources ready 的卡。
- committed fixture 与服务器实时 materialized 卡并存。
