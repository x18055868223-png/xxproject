# 缺口实现规范：先实现，后标定（Q2 答案）

> 目的：让四大缺口（及任何未标定因子）可以**先把结构实现并接入**，等真实数据到位再调试优化；但**设计与实现层面保持收束干净**——占位绝不污染判断、绝不变成技术债。
> 最后更新：2026-06-02（demo v0.1）

---

## 0. 核心矛盾与解法

- 矛盾：要「先实现」（结构完整、已接线）又要「不杂乱」（占位不能乱影响决策）。
- 解法：**统一因子接口 + 状态标记 + 安全默认 + 注册表登记 + 提升门槛**。占位因子在结构上是一等公民（已接线、有包字段、面板有位），在行为上是惰性安全的（不影响或只保守收紧决策），且全局一眼可辨（注册表 status）。

---

## 1. 统一因子接口（所有 31 因子同一形状）

每个因子是纯函数，返回统一 `FactorResult`（见 `shared/factor_spine.py`）：
```python
FactorResult(
  factor_id: str,            # 注册表主键，如 "X10_portfolio_budget"
  status: str,               # LIVE | PLACEHOLDER | WARMING | OFFLINE
  value,                     # 数值/裁决（域包据此收束）
  detail: dict,              # 可观测原始读数（面板/回放/归因用）
  calibration: dict,         # {required: bool, needs: "...", placeholder_values: {...}}
  reason_codes: list,        # 机器可读理由
)
```
域编排器只认这个形状，不关心因子内部怎么算。新增因子=新增一个返回 `FactorResult` 的函数 + 注册表一行。

---

## 2. 状态生命周期

```
PLACEHOLDER  ──(真实数据 + 标定 + 用户拍板)──▶  LIVE
   │                                              ▲
   └── WARMING（在线暖机，数据未够）───────────────┘
OFFLINE：只在回放/标定 harness 跑，不进 live tick（如全链回放）
```
- `PLACEHOLDER` = 已实现已接线、**未标定**，按「安全默认」行为。
- `WARMING` = 已 LIVE 逻辑，但运行时样本不足（如 CVD 滚动分位暖机），自动降权/降置信。
- `LIVE` = 已标定/已验证，正常影响决策。
- `OFFLINE` = 离线工具因子（回放期望），不在实时链。

**提升 PLACEHOLDER→LIVE 的硬门槛**（缺一不可）：
1. 有真实数据支撑的标定（非拍脑袋）。
2. 注册表 `status` 翻成 LIVE + 记录标定来源。
3. [实现流程与经验] 日志追加一行（谁/何时/依据/前后行为差异）。

---

## 3. 占位安全默认（收束干净的核心约定）

**占位因子绝不能悄悄放松任何门或夸大任何机会。** 二选一：

- **A. 中性无效（neutral）**：返回不影响决策的值（vote=0 / 权重=0 / 调制=1.0）。用于「方向/置信类」占位（如某证据未标定 → 不投票）。
- **B. 保守收紧（conservative-safe）**：只会 **挡单 / 缩量 / 提前退出**，绝不放行更多。用于「风控/门类」占位（如组合预算未标定 → 用保守硬上限宁可挡）。

> 铁律：**占位因子的误动方向只能是「更安全」**。组合预算占位 = 偏紧（易挡），止盈占位 = 偏早（易退），sizing 占位 = 偏小（test_size）。绝不允许占位因子让系统多交易、多下注、晚退出。

四缺口的占位安全默认：
| 缺口因子 | 占位类型 | 安全默认行为 |
|---|---|---|
| 组合硬预算 X10 | 保守收紧 | 并发/vega/gamma/保证金用保守硬上限，超即 size=0 |
| 区制sizing X11 | 保守收紧 | base=test_size；weak/VRP薄/GGR逆/vol扩张 → ×0.5 或 0 |
| 回撤熔断 X12 | 保守收紧 | 占位阈保守（小回撤即熔断 size=0）|
| 止盈 X13 | 保守收紧（偏早退） | 权利金回吐到保守目标比例即标 TAKE_PROFIT_READY |
| 末日Gamma退出 X14 | 保守收紧 | 剩余 DTE/gamma 占位阈偏早触发退出 |
| 时间退出 X15 | 中性观察 | 到阈值只标 REVIEW，不自动动作 |
| 最小归因 X20 | 中性观察 | 只记录分解，不进任何门（纯观测）|
| 全链回放 X21 | OFFLINE | 不进 live tick，只离线出期望 |

## 4. 收束干净·五不变量（违反即「乱」，必须挡回）

1. **一因子一模块一注册行一域**：没登记进注册表的因子不允许存在；不属于任何域的因子不允许存在。
2. **因子只经域包对外**：跨域不得直读因子内部状态；改因子只需守住域包字段契约。
3. **占位安全方向单调**：占位只能更安全（挡/缩/早退），绝不放松门或夸大机会。
4. **占位可见**：面板/日志/注册表必须显示因子 status；占位项标「未标定」，不得伪装成生产能力。
5. **提升留痕**：PLACEHOLDER→LIVE 必须真实数据标定 + 注册表翻态 + 经验日志留一行。

---

## 5. 与全链回放/归因的关系（让占位「可被真实数据优化」）

占位因子的 `calibration.needs` 字段声明它**需要什么真实数据才能标定**。两个数据底座供给：
- **AttributionPackage（X20）**：每笔交易扣成本分解（theta/方向/IV-RV/费用/滑点/保护腿/hedge/residual）——标定 VRP edge、sizing、止盈的依据。
- **ReplayExpectationPackage（X21）**：全链扣成本净期望分桶——标定阈值、验证「扣成本正期望」是否成立（闸 B 硬前提）。

因此「先实现后标定」不是空话：占位接线时就声明所需数据，归因/回放积累数据，再按提升门槛逐个翻 LIVE。**实现顺序与标定顺序解耦，但都收束在注册表里可追踪。**
