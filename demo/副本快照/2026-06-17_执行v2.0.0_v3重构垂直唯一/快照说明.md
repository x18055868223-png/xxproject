## 本次（v3 重构）关键变化
1. **垂直唯一**：彻底删除日历价差运行路径（MODE_CALENDAR/ENABLE_CALENDAR/残值摊销）。
2. **单一持续主链** `run_cycle` 取代旧 PLAN/ORDER 双脚本（旧式不再是运行入口）。
3. **交互层重做**：状态栏交互控制台 + 操作提示；FMZ 按钮 + **短确认码**硬授权（冻结审批快照 + 结构化幂等）；软授权止盈（与 position_id 绑定、非阻塞）。
4. **5 门控拆分**：`ALLOW_ENTRY/EXIT/HEDGE_TRADING` + `KILL_NEW_RISK` + `EMERGENCY_REDUCE_ONLY`（取代单一 ALLOW_TRADING）。
5. **信号→执行接收链**（执行侧）：`signal_receiver` 校验 schema/TTL/reject/data_quality + 降级（总线不可用禁新开、不阻断持仓管理）。
6. **受控真实开仓**：13 项预提交 + 投影预算真实算法(fail-closed) + 冻结入场利润上限。
7. **低成本止盈退出**：资格与成交解耦（保护腿价值不入分母）+ 预算反推价格上限 + 逐步买回；先平短腿→`SHORT_FLAT_LONG_RESIDUAL`→保护腿回收。
8. **BTC-PERPETUAL 对冲生命周期**：固定工具 + 目标随剩余短腿 + open/reduce 分离(reduce_only 仅减仓) + 孤儿紧急态。
9. **四输出动作仲裁**：退出不可执行可回退对冲。
