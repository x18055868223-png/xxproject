## 本次里程碑：信号层 v1.2.0
依据《信号审计推送格式重设计 v1.2》（`docs/`）落地。**只改推送渲染与可观测，不改方向/EDB 权重/阻断/执行契约。**

- **A 推送正文重设计**（`render_review_card_push`）：由旧的 `◎` 分段叙述改为「**头部 + 四审计层 + 复盘索引**」固定结构——
  - 头部：标题(含 v1.2 + 卡号) / 时间·现价 / 结论(lean·support·动作) / 置信·EDB·一致·覆盖 / 一句话。
  - 【一、背景层】NR 窗口·Anchor·TMV(24h/48h/blend/冲突)·Macro(score·regime·conf·组件 bps)。
  - 【二、修正层】Gamma(regime·net_gamma·pin·flip·ggr_mult·veto)·SRD(vote·RR·Δrr·data)·Funding(last·effect·hard)。
  - 【三、论证层】score_raw→gamma→EDB；`confidence = strength×agreement×coverage×gamma=final`；证据按贡献绝对值排序，一行一证据。
  - 【四、冲突层：为什么…】冲突 level·ratio / 同向 / 反向(带 vote·w) / 剔除·降权 / 阻断·解除条件。标题后缀按可交易/暂不交易/一票否决动态切换。
  - 【复盘索引】JSONL 卡号 + 字段指引 + 未校准免责声明。
  - 数值优先、缺失项显式 `N/A`、首屏先答「结论/置信/为何不交易」、单条推送、`fmz_push` 仍补末尾 ` @`。新增 formatter：`_operator_headline`/`_background_layer_lines`/`_correction_layer_lines`/`_reasoning_layer_lines`/`_conflict_layer_lines`/`_evidence_row`/`_compact_macro_components`/`_one_line_summary` 等；旧的 `◎` 渲染助手已删除。
- **B 推送自检开关**（新增全局变量 `signal_review_push_test`，默认 False）：为 True 时**启动即推送一次合成样例审计卡**（无需等信号生成），仅用于验证推送链路是否通 + 样式是否正确；推送带「**【推送自检·非真实信号】**」横幅，绝不会被误认为真实信号；幂等（一进程只推一次）；关时静默。样例卡由新增 `signal_review.build_sample_review_card()` 生成（WAIT_CONFIRMATION 情景，铺满全部四层）。

## 部署 / 使用
FMZ 控制台 → Python 策略 → 整份粘贴。信号默认只读；执行默认空跑。
- 验证推送是否打通 + 看样式：临时设 `signal_review_push_test=True`，重启策略 → 收到一条「推送自检」样例邮件/推送后，**改回 False**。
- 正常审计推送：设 `signal_review_push_enabled=True`（仅 `NR_REPAIR_CONFIRMED` 出信号时推一条）。
