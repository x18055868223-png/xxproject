## 1. 装配模型：执行会话作骨架，按域调真实因子

```
ExecutionSession (session_core, 复用)  ← 骨架/状态机/授权
  状态流: IDLE→SIGNAL_OBSERVED→PRICE_GATE→PLAN_READY→PLAN_LOCKED
          →APPROVAL_INTENT→ARMED_PREVIEW→ORDER_COMMITTING
          →POSITION_OPEN→POSITION_MANAGE‖HEDGE_WATCH→EXIT_OR_REUSE→CLOSED
  每段调用真实因子，不重写：
    PRICE_GATE      → vrp_adapter(复用) → 真实 VRP assess_window/candidate
    PLAN_READY      → 真实 plans.plan_assemble/plan_rank + leg_selection + spm_sim（已删 KPF）
    PORTFOLIO_BUDGET→ risk_controls.evaluate_portfolio_budget(复用, 占位安全)
    APPROVAL/ARMED  → session_core.lock_plan/approve(复用, plan_hash+TTL)
    ORDER           → 真实 execution.exec_open_structure（maker-only，ALLOW_TRADING 门）
    LEDGER          → 真实 ledger + EntryRiskAnchor（加 session/VRP 血缘，删 entry_kpf_buffer_state）
    POSITION_MANAGE → risk_controls.decide_position_manage(复用, 占位安全)
    HEDGE_WATCH     → 真实 hedge_risk.evaluate_position_risk（两项持续性，删 KPF）
    ATTRIBUTION     → risk_controls.build_attribution(复用, 占位观测)
```

要点：**复用模块负责"会话/门/缺口契约"，真实模块负责"选腿/报价/记账/对冲"**。两者经域包耦合，不互穿内部（收束架构 §1）。
