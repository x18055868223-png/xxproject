# 中性回路工程总纲 v2026.06.22-r3.0

本版本用于把当前 `xxproject` 工程快照推进到 r3.0。它是工程备份、审计和恢复入口，不是交易启用声明。

## 1. 当前系统边界

当前链路由五个主要部分组成：

1. **FMZ 信号层本体**：v1.4.0，只读观察行情与因子，生成短推、状态栏和 `signal_review.jsonl`。
2. **GEX Monitor API**：策略服务器上的 FastAPI 服务，提供清洗后的 `/v1/info`，包括 GEX、Gamma、IV/RV、P/C、flow 与本地历史 rank 分位。
3. **审计静态页面**：服务器静态页面读取 materializer 输出的 `signal_cards/`，展示 GEX/Gamma、rank、LLM 复核和审计证据。
4. **LLM 复核旁路**：Gemini 两段式盲审复核脚本读取真实信号审计卡，生成 sidecar，再由 materializer 合并进前端卡片。
5. **执行层交付物**：Deribit 垂直价差执行链 v2.7.0，当前作为 `DRY_RUN_CANDIDATE` 全空跑资产保留。

信号层与审计层已经进入服务器观察和排障链路；执行层仍需单独端到端空跑、Tier1 检查和人工授权门验证。

## 2. 信号层

入口文件：

```text
demo/最新交付物/neutral_regulation_demo_fmz.py
```

当前特征：

- `demo_version = 1.4.0`
- `schema_version = nrd.schema.v1.0.0`
- 默认只读观察，不选腿、不报价、不下单。
- 输出 FMZ 短推，完整证据写入 `signal_review.jsonl`。
- 内置 `session_context`、时区/前提耐久度、决策矩阵和前端审计字段。
- GEX 增强通过 `gex_info_base_url` + `gex_info_token` 读取服务器 `/v1/info`。
- LLM 不在 FMZ 主循环内调用；真实复核由服务器旁路完成。

## 3. 审计页面与 LLM 旁路

入口目录：

```text

```

服务器基线路径：

```text

/home/bitnami/fmz2/logs/storage/668422/demo/logs/signal_review.jsonl
/etc/signal-audit/llm.env
```

当前约束：

- materializer 读取 FMZ JSONL 与 `signal_llm_reviews.jsonl` sidecar，生成 `signal_cards/index.json`、单卡 JSON 和 `fallback.js`。
- LLM 复核只对真实信号审计卡运行，不修改 `decision`、`reasoning`、`blocking` 或交易许可。
- 前端关键指标和 FMZ 短推不展示旧的“置信未校准”残留；置信仍只是内部证据分，不是胜率。
- 周末真实降档逻辑本轮未修，仍是下一轮时区层清理 TODO。
