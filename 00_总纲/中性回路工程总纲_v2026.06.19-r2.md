# 中性回路工程总纲 v2026.06.19-r2

本版本用于把当前正在运行或即将观察的信号层链路统一收束到 `xxproject` 工程快照中。它不是交易启用声明，而是一次可恢复、可审计、可排障的工程整合。

## 1. 当前系统边界

当前链路由五个主要部分组成：

1. **FMZ 信号层本体**：在 FMZ 中只读观察行情与因子，生成信号、短推、状态栏和 `signal_review.jsonl`。
2. **GEX Monitor API**：策略服务器上的 FastAPI 服务，提供清洗后的 `/v1/info`，包括 GEX、Gamma、IV/RV、P/C、flow 与本地历史 rank 分位。
3. **审计静态页面**：服务器 Apache/Nginx 下的静态页面，读取 materializer 输出的 `signal_cards/`。
4. **LLM API 复核旁路**：Gemini 复核脚本读取真实信号审计卡，生成 sidecar，再由 materializer 合并进前端卡片。
5. **执行层交付物**：Deribit 垂直价差执行链，目前作为未正式启用的空跑/后续测试资产保留。

信号层与审计层已经进入服务器实测观察阶段；执行层仍需单独实盘模拟与授权门验证。

## 2. 信号层

入口文件：

```text
demo/最新交付物/neutral_regulation_demo_fmz.py
```

当前特征：

- `demo_version = 1.3.0`
- `schema_version = nrd.schema.v1.0.0`
- 默认只读观察，不下单。
- 输出短 FMZ 推送，完整证据写入 `signal_review.jsonl`。
- GEX 增强通过 `gex_info_base_url` + `gex_info_token` 读取服务器 `/v1/info`。
- GEX rank 只作为只读上下文和状态栏/审计页展示，不参与交易执行。
- LLM 字段在 FMZ 内保留为兼容配置，但主循环不进行 LLM HTTP 调用，避免阻塞 FMZ。

## 3. GEX Monitor API

入口目录：

```text
05_GEX监控API_数据增强接口/
```

当前职责：

- 抓取 GEX Monitor 公开页面可见指标。
- 输出 `/health`、`/v1/info`、`/v1/refresh`。
- `/v1/info` 包含：
  - `gex_board.total_net_gex`
  - `gex_board.dvol`
  - `gamma_exposure.flip_point / magnet_price / walls`
  - `volatility.iv_rv_ratio / pcr / term_structure`
  - `flow.call_put_bias / put_call_ratio`
  - `rank.window` 与 `rank.metrics`

rank 规则：

- 每次全量刷新追加一条轻量历史到本地 JSONL。
- 历史不足 30 天时使用已有样本计算分位。
- 历史超过 30 天时只用最近 30 天窗口计算当前分位。
- 全量历史保留，不因为滚动窗口删除。

## 4. 审计页面与 materializer

入口目录：

```text
deploy/signal_audit/
tools/materialize_signal_cards.py
```

服务器基线路径：

```text
/opt/signal-audit
/opt/signal-audit-tools
/home/bitnami/fmz2/logs/storage/668422/demo/logs/signal_review.jsonl
```

materializer 负责：

- 读取 FMZ `signal_review.jsonl`。
- 读取 LLM sidecar `signal_llm_reviews.jsonl`。
- 生成 `signal_cards/index.json`。
- 生成每张 `signal_cards/*.json`。
- 生成 `fallback.js`。
- 控制读取窗口，避免 1GB 服务器长期全量读大文件。

审计页面当前展示：

- Gamma/GEX 重点
- GEX rank 分位
- LLM 复核与理论主动倾向视角
- 程序化决策结论
- 证据账本、冲突、阻断、质量源

## 5. LLM 复核旁路

入口：

```text
tools/gemini_signal_llm_review.py
deploy/signal_audit/run_signal_llm_review.sh
deploy/signal_audit/signal-audit-llm-review.service
deploy/signal_audit/signal-audit-llm-review.timer
```

服务器 key 配置：

```text
/etc/signal-audit/llm.env
```

当前设计：

- 只对真实信号审计卡调用。
- 自检样本和普通观察轮次不调用。
- 输出 `llm_review`，不修改 `decision`、`reasoning`、`blocking`、`trade_allowed`。
- 增加 `theoretical_active_view`，作为非信号、非门控、非交易建议的理论主动倾向参考。
- 增加 `gamma_regime_lens`，用于分析 Gamma 体制对尾部、反身性和风险分布的影响。

sidecar 路径：

```text
/opt/signal-audit-tools/signal_llm_reviews.jsonl
```

- FMZ 是否继续正常短推和写入 JSONL。

- Gemini sidecar 是否只在真实信号后生成。
- 审计页面是否显示 LLM 复核和理论主动倾向。
- 自检脚本是否能在异常时指出具体模块。
