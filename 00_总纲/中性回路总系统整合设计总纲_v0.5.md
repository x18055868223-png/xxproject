## 0. 本轮变更（相对 v0.4）

1. **确定整合路线 = 执行会话式（方案 B）**：信号层独立只读；执行层升级为统一 `ExecutionSession`，VRP/账本/持仓风险/对冲并为其内部子域。长期可演进到方案 C（编排器），现在不推。
2. **消灭重启式 PLAN/ORDER 割裂**：引入会话内 `ApprovalIntent` 状态机（`plan_hash` + `TTL` + `approval_state`）取代「改 SELECTED_PLAN/ALLOW_TRADING → 重启 ORDER」。安全性不降反升（显式两步 + 重验 + 防重排），但需先做 FMZ 交互/持久化 spike。
3. **明确两层运行时拓扑**（信号/证据层 + 执行/风险层），但**三模块第一性不变**——三者作为执行层内部子域存在，不丢失各自第一性。
4. **新增四大缺口为开闸前硬前提**：①全链扣成本净 P&L 回放 ②组合硬风险预算 ③基础赢家管理（止盈/末日 Gamma 退出）④最小收益归因。它们补的是「polish 不只是把按钮做顺，而是让信号/成本/仓位/退出/归因成为同一会话里的可审计事实」。
5. **校正三处参考稿疏漏**（详见 §1）：
   - 参考稿落地分期**漏了 KPF 减法**——经 grep 实证（2026-06-02）KPF 仍在执行层/对冲代码 6 处，本稿明确**减法为路线第一步**。
   - 参考稿 §6 仍写对冲持续性读「KPF buffer 或其替代」——**校正为两项制 `{EDB_ADVERSE, GGR_ADVERSE}`**（v0.3 §3.3 已删 KPF）。
   - 参考稿偏执行体验、弱化信号层 P0 校准——**校正：信号 P0 校准（`nr_threshold_profile` relaxed_test→production）仍是闸 B 平行硬前提**。

## 4. 目标架构：两层运行时 + 三模块第一性子域

```text
┌─ 第一层 Signal / Evidence Layer（信号与前置证据层｜独立只读 FMZ）
│   中性回路 v0.5.4：时序(DIE+Anchor/NeutralRepair) + 方向(EDB) + 安全摘要(GGR/宏观/资金费率)
│   → SignalEvidencePackage
│       { timing_window · direction_evidence · strategy_recommendation
│         · pre_trade_context · data_quality · reject_state }
│
│   本机总线（loopback / 共享文件原子写 / 兜底最小 HTTP）；信号不出本机
▼
┌─ 第二层 Execution / Risk Layer（执行、账本、组合风险、持仓管理｜持钥 FMZ = ExecutionSession）
│   按子域组织，三模块第一性在此保持：
│   ├─ Pricing Gate Domain  : VRP 窗口门 + 候选门（建仓前定价硬门）
│   ├─ Plan Domain          : leg_selection / S:PM / cost（候选枚举与价值指标）
│   ├─ Approval Domain      : plan_lock / plan_hash / TTL / armed_preview（消灭重启割裂）
│   ├─ Order Domain         : maker-only / 保护腿优先 / 只追一步 / kill-switch
│   ├─ Ledger Domain        : 库存 / 状态机 / reconcile / EntryRiskAnchor
│   ├─ Portfolio Risk Domain: 并发/vega/gamma/保证金 硬预算 + 回撤熔断（缺口2）
│   ├─ Position Risk Domain : 对冲监控（touch drift / tail accel / exit friction）+ 赢家管理（缺口3）
│   └─ Attribution Domain   : theta/方向/IV-RV/费用/滑点/保护腿/hedge/residual（缺口4）
```

**数据流（会话化）：**
```text
SignalEvidencePackage(放行)
  → [ExecutionSession] IDLE → SIGNAL_OBSERVED
  → PRICE_GATE(VRP 窗口门→候选门，只跑 EDB 背书侧)
  → PLAN_READY(双门 PASS 候选 → S:PM/排序) → PLAN_LOCKED(plan_hash+TTL)
  → APPROVAL_INTENT → ARMED_PREVIEW(重取价/重验 VRP/S:PM/账本)
  → ORDER_COMMITTING(显式 ALLOW_TRADING 才真单) → POSITION_OPEN
  → POSITION_MANAGE(止盈/末日 Gamma/时间退出) ‖ HEDGE_WATCH(干跑对冲意图)
  → EXIT_OR_REUSE → CLOSED
  ↳ 全程以 session_id 串包；成交落 EntryRiskAnchor + 最小 AttributionPackage
```

新增原则（叠加 v0.4）：
- **VRP / EDB / GGR / 宏观独立 AND**：任一不过即不交易；VRP 肥不降 EDB 门。
- **组合硬预算是入场前的额外 AND 门**：单笔结构正确也可能因并发/尾部共振失控（缺口2）。
- **持仓后只做风险压缩与赢家保全，不重判方向、不加仓**。

## 5. 模块第一性边界（继承 v0.3 §5 + 参考稿 §2，校正）

### 5.1 应放入信号层 vs 执行层（参考稿 §2.5 表，校正后）

**信号层只产出「是否值得进入执行评估」的前置信息**：`timing_window`（DIE+Anchor 修复窗口/TTL）、`direction_evidence`（EDB lean/support/confidence/coverage/reason）、`strategy_recommendation`（24h/48h expiry + 策略侧，不选 strike）、`pre_trade_context`（GGR regime/veto + 宏观/资金费率摘要）、`data_quality`（新鲜度/缺字段/校准 profile/包版本）、`reject_state`（`NO_TRADE_BLOCKED`/`WAIT_CONFIRMATION` + 原因）。

**信号层不放**：VRP 定价门、Deribit 费率/价差/滑点/maker 判断、S:PM 模拟、具体腿、ApprovalIntent、账本/持仓/对冲/止盈/滚动、sizing/组合预算、交易后归因。理由：这些都依赖执行账户、实时盘口、账本或成交事实，放进信号层会把只读前置模型污染成半执行系统。

**执行层承接「从前置信息到真实交易事实」的全部内容**（见 §4 子域）。

### 5.2 各模块第一性（不变）
- **信号层**：窗口何时开 + 哪一侧收敛；不持钥/不选腿/不报价/不下单/不做持仓后对冲。
- **VRP**：被放行的 expiry/侧在当前实时定价下是否值得卖 vol；不判方向/不选期/不进主排序权重/不替代 S:PM/不覆盖 `NO_TRADE_BLOCKED`/不因肥降门。
- **执行层**：被允许的方向窗口里能否形成可复核/可记账/可控摩擦/可执行结构；授权后按 maker-only + 账本纪律执行；不重判方向/不绕安全门/不把风控变加仓。
- **对冲模块**：持仓后风险是否相对入场锚恶化、是否持续、期权退出是否比期货临时对冲更差；不入场/不重做 EDB/VRP/不把期货腿变方向交易/第一阶段只干跑。

## 6. 执行会话状态机与新增契约

### 6.1 主状态机（取代重启式 ROUND_MODE）
```text
IDLE → SIGNAL_OBSERVED → PRICE_GATE → PLAN_READY → PLAN_LOCKED
     → APPROVAL_INTENT → ARMED_PREVIEW → ORDER_COMMITTING
     → POSITION_OPEN → POSITION_MANAGE ‖ HEDGE_WATCH → EXIT_OR_REUSE → CLOSED
```
核心变化：`ROUND_MODE` 不再是用户主语（保留为内部/兜底）；`SELECTED_PLAN` 不再要求重启手填；方案编号是身份的一部分，**真正的安全锚点是 `ApprovalIntentPackage`（plan_hash + TTL + 显式 allow_real_order）**。

### 6.2 ApprovalIntentPackage（新增）
```json
{
  "schema_name": "ApprovalIntentPackage",
  "schema_version": "nrd.integration.approval_intent.v0.1",
  "session_id": "", "signal_package_id": "", "plan_id": 0, "plan_hash": "",
  "plan_created_ts": 0, "approval_created_ts": 0, "ttl_sec": 90,
  "approval_state": "LOCKED|ARMED|EXPIRED|CANCELLED|CONSUMED",
  "allow_real_order": false, "operator_note": "",
  "precommit_checks": {"signal_fresh": false, "vrp_rechecked": false,
    "spm_rechecked": false, "quotes_rechecked": false, "ledger_rechecked": false,
    "spread_ok": false, "maker_only": true}
}
```
`plan_hash` 防方案重排/刷新/同号碰撞；`ttl_sec` 防旧报价进入执行；`approval_state` 支持同一运行内撤销/过期/消费；`allow_real_order` 仍需显式打开，不因锁定方案自动下单。

### 6.3 ExecutionSessionPackage（新增，面板/日志/回放的中枢）
```json
{
  "schema_name": "ExecutionSessionPackage",
  "schema_version": "nrd.integration.execution_session.v0.1",
  "session_id": "", "state": "PLAN_READY",
  "signal": {}, "vrp_gate": {}, "plan_menu": [], "locked_plan": {},
  "approval_intent": {}, "order_intent": {}, "position_ledger": {},
  "position_risk": {}, "portfolio_budget": {}, "attribution": {}, "reason_codes": []
}
```
好处：UI 显示一条「执行故事」而非一堆孤立表；回放按 `session_id` 串起信号→VRP→计划→授权→下单→账本→对冲→归因；模块分开写，用户只看一个会话。

### 6.4 面板 polish（三段 cockpit）
1. **信号与资格**：signal_state / direction / freshness / reject reason。
2. **建仓计划**：VRP gate / 拒绝漏斗 / S:PM / plan menu / locked plan / cost / **组合预算占用**。
3. **持仓管理**：ledger / 库存 / short 状态 / position risk / **赢家管理动作** / hedge intent。
原则：主语是「当前会话状态」不是「计划轮/下单轮」；只显示下一步可做动作；每个拒绝落到一个明确门（信号/VRP/报价/S:PM/库存/授权/账本/预算/对冲）；`VRP edge` 与 `plan_ev` 并列必须标口径差异、不相加。

## 8. 缺口评估（四大缺口分级）

判据：是否影响开闸前正确性 / 是否影响第一性边界 / 是否影响 polish 必要闭环 / 是否需足够样本才能做对。

### 缺口 1 · 全链扣成本净 P&L 回放 —— **必须，闸前硬前提**
信号方向正确 ≠ 执行结构扣完成本后为正；VRP 只答「权利金够不够补偿」，不替代全链净 P&L。
- 做法：`SignalReplay → ExecutionCandidateReplay → VRP gate → S:PM/cost/accounting → net_pnl_after_fee_spread_slippage → attribution`。
- 最低验收：输出 `gross_credit/fee/spread_cost/chase_cost/protection_cost/net_credit/expiry_or_exit_pnl`，按信号强弱/方向/DTE/VRP gate/GGR regime 分桶；**报扣成本净期望 + 尾部亏损 + 回撤 + 样本数**，不只报胜率。该验证未完成前，只能说「执行链可控/可回放/可空跑」，**不能说「长期盈利已成立」**。

### 缺口 2 · 组合 sizing 与区制下注 —— **硬预算必须；连续 sizing 后置**
固定风险比例只是单笔口径。**近期必须**：并发持仓上限、单标的 short gamma/vega 上限、账户级最大保证金占用、日内/周内最大亏损熔断、连亏/波动扩张下的最小缩量、`ALLOW_TRADING` 开闸前的 `PortfolioRiskBudgetPackage`。**允许后置**：Kelly/分数 Kelly、regime→size 连续函数、跨策略 vega/gamma 配额、动态下注曲线（样本不足时易伪精确）。
- 推荐第一版（保守、确定性）：
```text
base = fixed_small_size
weak signal → ×0.5；VRP edge thin → ×0.5
GGR adverse / 趋势区制 / vol 扩张 → 0 或 min(size, test_size)
组合 gamma/vega/margin 预算超 → 0；回撤熔断 → 0
```

### 缺口 3 · 赢家管理 / 滚动 —— **止盈+末日 Gamma 退出必须；复杂 roll 后置**
现在只有尾部防守（对冲），无止盈/滚动；空 Gamma 不管理赢家会把到期前 gamma 尾部白敞着。**近期必须**：权利金回吐到目标比例止盈、末日 Gamma/触界概率升高即退出（即使未亏）、剩余 DTE 阈值强制复核、保护腿复用前重验（不因有库存强卖新 short）。**允许后置**：多轮滚动优化、自动 roll 候选搜索、基于 term structure 的动态 roll、保护腿跨周期最优摊销。
- 第一版管理态：`TAKE_PROFIT_READY → TIME_EXIT_REVIEW → GAMMA_DECAY_EXIT → ROLL_CANDIDATE_DRYRUN → EXIT_OR_REUSE`。

### 缺口 4 · 收益归因闭环 —— **最小版必须**
无归因则分不清赚的是 theta / 方向 / IV-RV 补偿 / 保护腿残值 / 运气；VRP edge 验证尤其依赖归因。**近期必须**：每笔结束写 `AttributionPackage`（最小字段：`theta_capture/directional_pnl/iv_rv_edge_proxy/fee_cost/spread_slippage_cost/protection_cost_or_recovery/margin_release_context/hedge_pnl/unexplained_residual`），与 `ExecutionSessionPackage` 绑定。**允许后置**：精细 Greeks 级归因、波动曲面分解、多日滚动归因图、归因驱动自动调参。

### 优先级汇总
| 缺口 | 开闸前必须 | 允许后置 |
| --- | --- | --- |
| 1 全链扣成本净 P&L 回放 | **是** | 仅更复杂多因子研究 |
| 2 组合 sizing/区制下注 | 硬预算**是** | Kelly/连续 sizing/组合优化 |
| 3 赢家管理/滚动 | 止盈+末日 Gamma**是** | 复杂自动 roll |
| 4 收益归因 | 最小版**是** | 高级 Greeks/曲面归因 |

**不能后置到实盘之后的四件事**：①全链扣成本净 P&L 回放 ②组合硬风险预算 ③基础止盈/末日 Gamma 退出 ④最小收益归因。

**Phase 1 · 减法落地（纯删除，先做）**：执行层删 `PLAN_WEIGHTS["kpf"]` 并归一 `{win_rate:0.375, rr:0.375, signal:0.25}`、`plans.py/leg_selection.py/strategy.py/accounting.py` 去 kpf；对冲删 KPF 持续性项与 `entry_kpf_buffer_state`、持续性重标定两项（0→LOW/1→MEDIUM/2→HIGH）、采纳 v1.0 收紧状态机、升 `position_risk.v0.4`；文档卫生。验收：回归 + 离线 fixture 全绿、对冲回放错配率不劣化、`HEDGE_READY` 更难触发。

**Phase 2 · 执行会话体验重构（不接 VRP 下单）**：新增会话内 `PLAN_LOCKED / APPROVAL_INTENT / ARMED_PREVIEW`，替代重启式 `ROUND_MODE=ORDER + SELECTED_PLAN`；仍 `ALLOW_TRADING=False`。**前置 spike：验证 FMZ 同进程交互/持久化能力**（命令栏锁定方案 / `_G` 存 ApprovalIntent / TTL 过期回退）。验收：同一运行会话锁定方案、plan_hash+TTL 生效、TTL 过期退回 PLAN_READY、方案刷新 hash 不一致不可执行、面板不再以「重启进下单轮」为主流程。

**Phase 3 · VRP 只读嵌入 PLAN + 收口 canonical**：按 `VRP执行层整合收口契约_v1.1.md` 把 4 原语收口（`normalise_iv→hedge_risk._normalise_iv`、`_norm_cdf→hedge_risk._norm_cdf`、`_option_fee→accounting.acct_option_fee_ccy`、`_spread_half_cost→accounting.acct_spread_cost`），保留 BS pricer 为执行层共享原语；窗口门→候选门漏斗，只跑 EDB 背书侧。验收：VRP BLOCK 候选不进可锁定方案、VRP 不进主排序权重、空跑不产订单、tests 覆盖 window/candidate/数据降级/EntryRiskAnchor 映射。

**Phase 4 · 账本/锚点增强 + 最小归因（缺口4）**：成交后 `EntryRiskAnchor` 写 VRP 血缘；账本记 `signal_package_id/plan_hash/approval_id` 形成 session replay 线索；每笔结束写最小 `AttributionPackage`。验收：每笔 short 有不可覆盖锚点、可追溯信号/VRP/计划/授权/报价/S:PM、对冲只读锚点不反做 VRP。

**Phase 6 · 全链扣成本净 P&L 回放（缺口1，闸前硬前提）**：`ReplayExpectationPackage` 全链回放，分桶报扣成本净期望/尾部/回撤/样本数。验收：给出真实分桶净期望，未为正不得进闸 B。

## 11. 不变量（合并 v0.3 §11 + 参考稿 §11，校正）

1. 信号层不持钥、不下单、不选腿、不读账本。
2. VRP 不进信号层、不降方向门槛、是硬门不是软权重；不覆盖 `NO_TRADE_BLOCKED`。
3. 方向只来自 `SignalEvidencePackage`（EDB）或人工明确配置；执行层与对冲都不自行推断方向。
4. 执行才是唯一期权下单点：不裸卖、保护腿在卖方腿外侧、固定风险比例、硬门控不因错过行情放宽。
5. **每次执行前必须重取价、重验 VRP、重验 S:PM、重验账本**；任何 approval 有 TTL，过期不可执行；任何方案用 `plan_hash` 防重排误选。
6. GGR 否决 / 宏观·资金硬阻断 / DIE+Anchor 窗口未开 → 执行层不得越过信号层「不可交易」结论。
7. 对冲只在持仓后运行、不入场、不自动改期权账本（只追加独立 hedge 账本）；持续性仅由 `{EDB_ADVERSE, GGR_ADVERSE}` 两项算；`HEDGE_READY` 必须同时满足风险+持续性+摩擦优势；对冲实盘开关独立于期权开仓开关。
8. 组合硬预算/回撤熔断超限 → size=0，单笔正确不豁免组合风险。
9. 所有模块通过版本化包互联、以 `session_id` 串包，不互相读内部状态；自动化前先有可回放输入/输出包且可人工复核。
10. schema 边界：信号层仍 `nrd.schema.v0.5.4`，整合通过消费其输出实现，不往其 payload 塞执行字段。
11. **未完成全链扣成本净 P&L 回放前，不得宣称「长期盈利已成立」**；VRP edge 在多时点 IV 验证前标 blocked。

1. 能说清两层 + 三模块第一性子域，且与最新代码版本一致。
2. KPF 减法落地（执行层 + 对冲），全链无 KPF 残留。

8. 保留全部正式拒绝路径（No-Trade/等待确认/执行不可行/期权退出困难/对冲不成立/预算超限/授权过期）。

## 15. 一句话总纲

中性回路总系统 = **证据驱动、会话连续的分段式卖方风险压缩流程**：
**① 信号/证据层**只读地确认窗口与方向证据 → **② 执行/风险层（一个 ExecutionSession）** 依次走 VRP 建仓前定价硬门 → 计划与 S:PM → 会话内授权（plan_hash+TTL，非重启）→ maker-only 下单 → 账本/入场锚 → 组合硬预算 → 持仓后赢家管理与干跑对冲 → 最小归因，全程以 `session_id` 可回放。
三模块第一性不变、安全门显式、证据可追溯、执行可撤销；**先把信号/成本/仓位/退出/归因做成同一会话里的可审计事实并跑出扣成本正期望，再小额开闸**。VRP/EDB/GGR/宏观/组合预算是独立 AND 硬门，口径全来自实时市场，故与「取消 KPF/SLRP」一致——不再造空间层，只量市场定价本身够不够补偿。

> 设计气质：**模块独立，链路连续；安全门显式，体验不割裂；证据可追溯，执行可撤销；先空跑闭环+扣成本正期望，再小额开闸。**
