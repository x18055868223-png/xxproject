# 中性回路工程总纲 v2026.06.18-r1

版本标记：`NRD-XXPROJECT-BACKUP-2026.06.18-r1`

本文是当前整合工程的工程级入口，用于替代散落文档中的过期路线描述。后续更新应优先改本文和 `99_工程资产索引/README.md`，再同步模块 README。

## 一句话结构

中性回路工程是一个分层的信号到执行风险系统：

1. 信号层只负责公开行情、期权/GEX/宏观/主动流证据合成，输出方向证据、审计 JSON 和短推。
2. VRP 门只做建仓前定价过滤，判断同一信号窗口下卖方权利金是否有足够净边际。
3. 执行层把通过信号与定价过滤的窗口转成可复核的垂直价差计划，默认空跑。
4. 对冲模块只在持仓后观察尾部恶化并生成意图，默认不真实对冲。
5. GEX API 是外置数据增强接口，给信号层提供清洗后的 `/v1/info` 软证据。

## 数据流

```text
公开行情 / Deribit / GEX API / Macro
  -> 信号层 FMZ 只读观察
  -> signal_review.jsonl
  -> materialize_signal_cards.py
  -> 静态审计页 signal_cards/index.json + 单卡 JSON

信号证据包
  -> VRP 建仓前定价过滤
  -> 执行层 PLAN / 空跑预览
  -> 真实交易门控保持关闭，等待独立上线验证
```

## 当前部署边界

- 信号层可以用于只读观察和审计数据积累。
- 审计页面可以读取 `signal_review.jsonl` 物化后的卡片。
- GEX API 可作为信号层软证据，缺失或 stale 只降级，不应解锁交易。
- 执行层、对冲模块仍不应开启真实交易，除非另行完成真实验证和门控审批。
