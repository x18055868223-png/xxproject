# 中性回路总系统整合设计总纲 v0.4（并入 VRP 建仓前定价门）

## 0. 本轮变更（相对 v0.3）

唯一结构性变更：**在执行层（②）建仓前补一道独立硬门 VRP（权利金风险补偿门）**。它回答 v0.3 三模块都没回答的卖方主问——「这个被信号层放行的到期窗口/卖出侧，现在是否值得卖波动率」。

- VRP 已 **v1.1 阶段性封版**（`系统总纲\VRP\`，见 `docs\VRP阶段性封版说明_v1.1.md`）。封的是安全门 + 回放机具 + 整合收口契约；**未声称卖方 edge**（待多时点 IV 前向采集）。
- VRP **只过滤、不判方向、不选期、不进主排序权重、不解 `ALLOW_TRADING`**；与 EDB/GGR/宏观是独立 AND 双门。
- 落点 = 执行层 PLAN 轮（不是信号层），按 `docs\VRP执行层整合收口契约_v1.1.md` 收口到执行层 canonical 原语。

为什么 VRP 不违背 v0.3「取消 KPF/SLRP」的口径：KPF/SLRP 试图在市场定价之外**再造一层空间证据**（被证无样本外边际）；VRP 相反，是**直接量市场定价本身够不够贵**（可执行 IV vs forward 波动门槛，扣完整摩擦），用的全是 Deribit 实时期权链 + 自算 RV，无外置重数据。它补的是 v0.3 留下的真实缺口（执行层有 `plan_ev` 但用 delta 当胜率 ≈ 风险中性、近同义反复，缺「权利金贵不贵」的物理补偿判断）。

## 2. 目标架构（刷新 v0.3 §4 数据流，插入 VRP 门）

```text
Public Market Data
  → ① 信号 FMZ（中性回路 v0.5.4，只读）
       → SignalEvidencePackage（时序+方向 EDB+GGR；DIRECTION_BIAS 定侧）

SignalEvidencePackage(放行) + Deribit 实时期权链(IV/Greeks) + 自算 RV(perp 1h)
  → ② 执行 FMZ · PLAN
       ├─[VRP 窗口门] 对每个 EDB 背书 expiry/侧：assess_window
       │     BLOCK/DISTORTED_REVIEW → 该 expiry 不进枚举
       ├─ 只在 PASS 窗口枚举候选（DTE/Delta 硬范围不变）
       ├─[VRP 候选门] 每条垂直候选：assess_candidate（扣 full-burn 净 edge）
       │     BLOCK → 剔除该候选
       └─ 双门 PASS 候选 → S:PM/既有 PLAN 排序
       → ExecutionPlanPackage（含 vrp_* 字段）

Approved → ② ORDER → PositionLedgerPackage + EntryRiskAnchor（含 VRP 入场血缘）
  → ③ 对冲模块（驻执行 FMZ）→ PositionRiskPackage / HedgeIntentPackage（DRY_INTENT_ONLY）
```

新增原则（叠加 v0.3 §4）：
- **VRP 与 EDB/GGR/宏观独立 AND**：任一不过即不交易；VRP 肥不降 EDB 门（肥权利金常是恐惧的价格）。
- **VRP 只过滤不选期**：多窗口 PASS 时由既有 PLAN 排序选，VRP 不抢期号。
- **VRP 只跑 EDB 背书侧**，不双侧评估。
- 空间/定价口径仍统一来自 Deribit 实时链 + 自算 RV，无外置重数据。

## 4. 契约（在 v0.3 §6 增 VrpGatePackage；ExecutionPlanPackage/EntryRiskAnchor 加 VRP 字段）

### 4.1 VrpGatePackage（来源：执行层 PLAN 轮，封版 v1.1）
窗口层 `WindowAssessment`（关键字段）：`window_id / expiry / side / front_anchor_iv / term_reference_iv_5_10d / front_to_term_state / rv_regime_anchor / rv_percentile / forward_vol_hurdle / representative_vol_edge / window_vrp_gate(PASS|BLOCK|DISTORTED_REVIEW)`。
候选层 `CandidateAssessment`：`window_id / side / short_delta / executable_short_iv / forward_vol_hurdle / vertical_net_credit_at_executable_quotes / vertical_net_credit_at_forward_vol_hurdle / full_round_trip_friction / candidate_vrp_edge_ccy / candidate_vrp_gate / vrp_residual_score / reason_codes`。
`forward_vol_hurdle = rv_regime_anchor × percentile_adjustment × cold_start_multiplier`（标量）；期限倒挂/加速/数据质量走 `DISTORTED_REVIEW` 路由，不进 hurdle 标量。

### 4.2 ExecutionPlanPackage（加候选级 VRP 字段）
`menu` 候选新增：`vrp_window_id / window_vrp_gate / forward_vol_hurdle / candidate_vrp_gate / candidate_vrp_edge_ccy / full_round_trip_friction / vrp_reason_codes`。

### 4.3 EntryRiskAnchor（加 VRP 入场血缘，与对冲共基线）
`entry_vrp_window_id / entry_executable_short_iv / entry_forward_vol_hurdle / entry_candidate_vrp_edge_ccy / entry_vrp_reason_codes`。对冲模块读此血缘、不反向重做 VRP。

### 4.4 排序权重（澄清 v0.3 §6.2 的 PLAN_WEIGHTS）
v0.3 已把 `kpf` 去掉、归一为 `{win_rate:0.375, rr:0.375, signal:0.25}`。**VRP v1 不进此权重**：`vrp_residual_score` 只作展示/tie-break/回放特征。理由：`rr`/`win_rate` 已与 IV 重叠，VRP 再进权重=同一信息多次计权。VRP 的价值在那道**硬门**，不在软排。面板上 VRP edge 须与 `plan_ev`（风险中性、近 0）区分标注，不可相加。

## 7. VRP 已验证 / 未验证（诚实边界，引封版 §3/§4）

- **已验证（真实数据）**：268,800 次压力遍历选出保守策略（危险/冷启动/薄 edge 通过=0）；90 天真实路径 leak-guarded 回放证实——低分位扩张不对称真实（realized/anchor 1.26 vs 高分位 0.68）、低分位+冷启动守护显著降低 hurdle 击穿（如 72h 低桶 0.39 vs 无调节 0.79）、hurdle 平均保护性（mean realized/hurdle≈0.88<1）。
- **未验证 / 推迟**：卖方 IV-vs-RV edge 未证，blocked on 多时点期权 IV（须前向采集）；所有阈值 PLACEHOLDER。

## 8. 一句话总纲（刷新 v0.3 §15）

中性回路总系统 = 「证据驱动的分段式卖方风险压缩流程」：**① 中性回路确认窗口与方向；②-门 VRP 在建仓前确认这个被放行的窗口/侧在 Deribit 实时 IV 上是否提供够厚、扣完整摩擦后仍为正的净权利金（不够贵就 no-trade）；② 执行层把通过双门的方向在 DTE/Delta 硬范围内转成可复核、可空跑、可记账的带保护腿结构；③ 对冲模块只在持仓后尾部恶化且期权退出更差时给干跑期货腿压尾意图。** VRP 与信号/GGR/宏观是独立 AND 硬门、只过滤不抢权重，口径全来自实时市场，故与「取消 KPF/SLRP」一致——它不再造空间层，只量市场定价本身够不够补偿。VRP 现为 v1.1 阶段性封版（安全门+回放机具已验，卖方 edge 待多时点 IV 验证）。
