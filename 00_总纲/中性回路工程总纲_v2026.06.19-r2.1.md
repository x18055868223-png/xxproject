# 中性回路工程总纲 v2026.06.19-r2.1

版本标记：`NRD-XXPROJECT-BACKUP-2026.06.19-r2.1`

本版本是 r2 的文档核验与版本锚定更新，不表示交易启用，也不改变信号层、GEX API、审计前端、LLM sidecar 或执行层代码行为。本文以当前代码资产和 2026-06-19 服务器实测链路为准，修正文档中旧版本号和部署口径容易误读的部分。

## 1. 当前事实源

当前运行链路由五类资产组成：

1. **FMZ 信号层本体**：FMZ 内只读观察，生成短推、状态栏、图表线和 `signal_review.jsonl`。
2. **GEX Monitor API**：策略服务器上的 FastAPI 服务，提供 `/v1/info`，含 netGEX、DVOL、IV/RV、P/C、flow 和 rank 分位。
3. **审计静态页面**：Apache/Bitnami 当前对外提供 `/signal-audit/`，读取 materializer 输出的 `signal_cards/`。
4. **Gemini LLM 复核旁路**：策略外 systemd timer 读取真实信号卡，生成 `signal_llm_reviews.jsonl`，再由 materializer 合并到审计卡。
5. **执行层交付物**：Deribit 垂直价差执行链，当前只作为全空跑/后续测试资产保留，未正式启用。

重要边界：

- FMZ 信号层不在主循环内调用 LLM API。
- LLM 复核不改变 `decision`、`reasoning`、`blocking`、`trade_allowed` 或执行门。
- GEX rank、LLM 复核和审计前端展示均属于观察/审计增强，不是交易授权。
- 执行层所有交易门默认关闭，文档备份不能等同于实盘启用。


当前职责：

- 输出 FMZ 简短推送，完整证据写入本地 JSONL。
- 在状态栏展示 GEX Monitor 数据增强，包括 rank 分位。
- 在策略图表中展示价格、Anchor、Gamma 相关线条。
- 构造 LLM 复核输入包字段，但不在 FMZ 主策略中执行 LLM HTTP 调用。

已退役口径：

- v1.2/v1.2.1 的 FMZ 长文/紧凑四层推送渲染器不再作为运行路径。
- `audit_archive/` 旧 scaffold 不是当前审计页面事实源。
- `signal_review_push_enabled` 和 `signal_review_push_test` 默认仍为 `False`，自检后必须改回关闭。

## 5. GEX Monitor API

入口目录：

```text
05_GEX监控API_数据增强接口/
```

当前职责：

- 提供 `/health`、`/v1/info`、`/v1/refresh`。
- 输出 `gex_board`、`gamma_exposure`、`volatility`、`flow`、`rank`、`sections`。
- 每次刷新追加轻量历史到本地 JSONL。
- 不满 30 天时用已有样本算 rank；超过 30 天时保留全量历史，但仅用最近 30 天计算当前分位。

rank 解释边界：

- `rank_pct` 是当前数值在滚动样本窗口内的相对位置，不是胜率。
- `quality=warming_up` 表示样本仍处于冷启动或未满月阶段。
- `total_net_gex` 同时提供方向分位与绝对值分位，前端应保留语义区别。

## 6. 审计页面与 materializer

入口：

```text
deploy/signal_audit/
tools/materialize_signal_cards.py
```

当前 contract：

- 输入：FMZ `signal_review.jsonl`。
- 可选输入：Gemini sidecar `signal_llm_reviews.jsonl`。
- 输出：`signal_cards/index.json`、单卡 JSON、`fallback.js`。
- 默认只读取尾部窗口，避免 1GB 服务器长期全量读大文件。

前端内容规划：

- 程序化结论、GEX/Gamma、rank、质量源、阻断、证据账本和 LLM 复核分区展示。
- 若 LLM 已列举同质化关键证据，前端应减少上方重复解释。
- 中文语义优化属于后续审计页面变更的默认要求。

## 7. LLM 复核旁路

入口：

```text
tools/gemini_signal_llm_review.py
deploy/signal_audit/run_signal_llm_review.sh
deploy/signal_audit/signal-audit-llm-review.service
deploy/signal_audit/signal-audit-llm-review.timer
```

服务器 key 配置：

```text
/etc/signal-audit/llm.env
```

当前设计：

- 默认模型：`gemini-3.5-flash`。
- 输出 schema：`signal_llm_review@1.2.0`。
- prompt 版本：`gemini_signal_review_prompt@1.2.0`。
- 只对真实信号卡复核；自检样本和观察轮次不应触发真实调用。
- 输出 `summary_cn`、`agreement_with_system`、`main_supporting_factors`、`main_risks_or_conflicts`、`operator_focus`、`invalid_if`、`gamma_regime_lens`、`theoretical_active_view` 等字段。
- `theoretical_active_view` 是理论主动倾向参考，不是下单信号、不是阻断、不是系统模型重算。
