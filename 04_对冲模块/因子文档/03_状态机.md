## 1. 一句话定位
把五决策变量 + persistence + 摩擦合成一个尾部风险态；仅 `HEDGE_READY` 时产出干跑期货腿对冲意图。

## 2. 当前具体实现（`_state_from_inputs`）
六态：`NORMAL / WATCH / EXIT_PREFERRED / HEDGE_READY / HEDGE_ACTIVE / MANUAL_REVIEW`。

```
existing_hedge                                   → HEDGE_ACTIVE
risk_elevated = p_now≥0.50 或 drift≥0.10 或 slope≥0.20
  not risk_elevated                              → NORMAL
risk_severe  = p_now≥0.65 或 drift≥0.20 或 slope≥0.30
  risk_severe 且 persistence∈{MEDIUM,HIGH} 且 hedge_friction_advantage → HEDGE_READY
  not hedge_friction_advantage                   → EXIT_PREFERRED
  其余                                            → WATCH
```
- `MANUAL_REVIEW`：方向非法 / 缺 EntryRiskAnchor（`_manual_review_package`）。
- `HEDGE_READY` → `_make_hedge_intent`：`hedge_side`（SHORT_CALL→BUY_FUTURE_OR_PERP / SHORT_PUT→SELL）、`hedge_size_mode`（FULL 0.90 / MEDIUM 0.60 / LIGHT 0.30，按 `_hedge_size_mode`）、`execution_mode=DRY_INTENT_ONLY`。

## 3. 关键阈值（现值，硬编码）
risk_elevated `p≥0.50 / drift≥0.10 / slope≥0.20`；risk_severe `p≥0.65 / drift≥0.20 / slope≥0.30`（`hedge_risk.py:273-278`）；size FULL 触发 `breached 或 p≥0.80`（`_hedge_size_mode:293`）。

## 4. 整合中的路径修改（v1.0 收紧）
采纳 v1.0 收紧项（总纲 v0.3 §3.3.3）：
- `WATCH` 设为"风险上升但尚不可对冲"**独立通道**，避免无持续性的 drift 直接变对冲意图（当前已基本如此：无摩擦占优→EXIT_PREFERRED，否则 WATCH）。
- `EXIT_PREFERRED` 仅风险已升且期权退出仍可接受时出现。
- hard boundary 击穿 / 极高 `p_touch` 在 persistence≥MEDIUM 且摩擦占优时直达 `HEDGE_READY`。
- 升 `position_risk.v0.4`；persistence 改两项制（见 02）。

## 5. 当前目标 / 待办
- 12 万样例回放确认 `HEDGE_READY` 更难触发（当前错配率 13.3%，目标不劣化）。
- 闸 3 前对冲始终 `DRY_INTENT_ONLY`。

## 6. 边界与陷阱
- `HEDGE_READY` = **风险硬成立 + 持续性≥MEDIUM + 期货摩擦占优**三者同时，是最难触发态，绝不可放松。
- 尺寸只到 LIGHT/MEDIUM/FULL 封顶比例，**不追求完全 delta neutral**。
- 状态机不入场、不判方向、不反手、不自动改期权账本——只追加独立 hedge 账本（整合不变量 §11.5）。
