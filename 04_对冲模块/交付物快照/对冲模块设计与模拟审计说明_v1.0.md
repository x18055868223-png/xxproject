# 对冲模块设计与模拟审计说明 v1.0

生成时间：2026-06-01  
适用系统：中性回路信号层 + KPF 空间层 + Deribit 期权执行层  
交付定位：说明本轮对冲模块需求拆解、批量模拟审计、规则收敛与 v1.0 建议方案；不是入场模型，不是方向交易系统，也不是实盘执行批准文件。

## 1. 本轮做了什么

本轮围绕“期权腿持仓后如何评估损失触界风险并决定是否用期货腿压尾部”做了四件事：

1. 把对冲模块从入场、方向判断、KPF 预测等语义中拆出来，只保留持仓后风险压缩职责。
2. 基于现有 Deribit 执行层 v0.3 对冲风险实现，构造 120,000 个合成持仓后样例做批量审计。
3. 从错分样例中识别过敏、迟钝和过早对冲的状态转移问题。
4. 形成一版 v1.0 规则建议，用于后续干跑、回放和小额实盘验证。

> 对冲模块不预测行情，也不重做 EDB。它只在期权腿已经建立后，判断损失触界风险是否相对建仓时恶化、恶化是否有持续证据、期权退出是否比期货对冲更差；只有三者同时成立，才生成期货腿干跑对冲意图。

基于本轮模拟审计，v1.0 建议把监控收敛为五个决策变量：

1. `touch_probability_now`：当前到期前触碰损失边界的市场隐含估计。
2. `touch_probability_drift`：当前触界概率相对建仓锚点的恶化。
3. `recent_deterioration_slope`：最近约 30 分钟的恶化速度。
4. `tail_exposure_acceleration`：净 delta/gamma 是否进入非线性加速区。
5. `exit_vs_hedge_friction`：期权退出摩擦是否显著高于期货对冲摩擦。

其他信息只允许成为上述变量的解释或确认项，不允许再扩展成一堆并列打分器。

## 2. 海量案例模拟审计

为避免设计只迎合 prompt，本次对当前 v0.3 规则做了 120,000 个合成持仓后场景模拟。

模拟覆盖：

- SHORT_CALL / SHORT_PUT 两个方向。
- 建仓后价格远离、接近、触及、穿透损失边界。
- IV、DTE、short delta、short gamma 的多区间变化。
- EDB 反向/同向/中性，含 confidence 与 coverage。
- GGR 正 Gamma pin、负 Gamma amplification、flip 附近/远离。
- KPF 缓冲层 intact、touched、penetrated、absent。
- 期权退出摩擦与期货对冲摩擦组合。

模拟参数：

| 项目 | 数值 |
| --- | --- |
| 样例数 | 120,000 |
| 随机种子 | 20260601 |
| 样例输出 | `交付物/hedge_policy_simulation/hedge_policy_mismatch_cases.csv` |
| 汇总输出 | `交付物/hedge_policy_simulation/hedge_policy_simulation_summary.json` |

v0.3 当前规则与更严格审计路线的主要差异：

| 差异类型 | 数量 | 含义 |
| --- | ---: | --- |
| `NORMAL -> WATCH` | 5,242 | WATCH 过敏，部分高概率但未继续恶化的场景被误认为需要观察升级 |
| `NORMAL -> EXIT_PREFERRED` | 4,880 | EXIT_PREFERRED 过早，风险未明确恶化时不应建议退出优先 |
| `WATCH -> HEDGE_READY` | 5,058 | 对冲过早，WATCH 阶段不应因为摩擦优势直接进入对冲 |
| `WATCH -> EXIT_PREFERRED` | 602 | 中等风险被过早归入退出优先 |
| `HEDGE_READY -> WATCH` | 11 | 极少数硬风险路径需要更直接的 HEDGE_READY 通道 |

审计结论：

1. v0.3 的方向是正确的，最小决策脊柱成立。
2. v0.3 的状态转移需要收紧，尤其是 HEDGE_READY 必须更难触发。
3. 摩擦不应提前改变风险状态，只能在风险成立后决定“退出优先还是对冲优先”。
4. KPF 缓冲层只能确认持续性，不能改变市场触界概率。
5. 高触界概率但无继续恶化证据，不应直接对冲，应进入 WATCH 或 MANUAL_REVIEW。

## 3. 第一性边界

对冲模块只回答三个问题：

1. **风险是否恶化**：现在比建仓时更容易触碰损失边界吗？
2. **恶化是否持续**：恶化有 EDB/GGR/KPF 证据确认吗？
3. **是否需要期货腿**：如果期权退出差，期货腿是否是更低摩擦的尾部压平工具？

它明确不做以下事情：

- 不决定是否入场。
- 不重新判断方向。
- 不把 KPF 写成概率、胜率或到达预测器。
- 不把期货腿变成反手交易。
- 不追求完全 delta neutral。
- 不在第一版自动实盘下期货单。

## 4. 核心输入

### 4.1 EntryRiskAnchor

每个 short leg 真实成交后必须记录入场风险锚点：

```json
{
  "entry_price": 0,
  "entry_dte_hours": 0,
  "entry_short_delta": 0,
  "entry_short_gamma": 0,
  "entry_iv": 0,
  "entry_loss_boundary": 0,
  "entry_touch_probability": 0,
  "entry_probability_confidence": "HIGH|LOW",
  "entry_boundary_distance_pct": 0,
  "entry_edb_side": "",
  "entry_gamma_regime": "",
  "entry_kpf_buffer_state": ""
}
```

原则：

- `entry_loss_boundary` 第一版使用该结构的 breakeven。
- 若 IV 可用，用 IV 触界模型。
- 若 IV 不可用，用短腿 delta 保守兜底，并标记 `LOW` confidence。
- 入场锚点不得被后续重算覆盖。

### 4.2 CurrentRiskSnapshot

每轮监控读取：

- 当前 price / DTE / IV / short delta / short gamma。
- 当前 `loss_boundary`。
- 过去约 30 分钟风险快照，用于恶化斜率。
- EDB aggregate：`lean`、`confidence`、`coverage`、`support_label`。
- GGR exception：`regime`、`distance_to_flip_pct`、`veto`、pin 信息。
- KPF buffer：按实时价格重锚后的不利方向缓冲层状态。
- option exit friction 与 futures hedge friction。

## 5. 概率模型

### 5.1 触界概率

触界概率是风控估计，不是胜率承诺。

首选方法：

```text
使用当前价格 S、损失边界 B、剩余到期 T、隐含波动率 sigma
估算到期前首次触碰 B 的概率。
```

SHORT_CALL：

```text
boundary_distance = ln(B / S)
```

SHORT_PUT：

```text
boundary_distance = ln(S / B)
```

触界近似：

```text
p_touch = clamp(2 * (1 - N(boundary_distance / (sigma * sqrt(T)))), 0, 0.98)
```

兜底方法：

```text
p_touch = clamp(abs(short_delta) * 1.8, 0, 0.95)
```

兜底方法只能用于缺 IV 或 IV 异常，输出必须标为低置信。

### 5.2 恶化度量

```text
touch_probability_drift =
  touch_probability_now - entry_touch_probability
```

```text
recent_deterioration_slope =
  (touch_probability_now - probability_30m_ago) / 0.5h
```

只使用一个最近窗口，避免三窗口模型引入解释噪音。

## 6. 持续性确认

持续性只用三项确认，不再把 TMV/CVD/Macro/Funding/SRD/GGR 全部拆开双算。TMV、CVD、Macro、Funding、SRD 已在 EDB 内综合。

### 6.1 EDB Adverse

成立条件：

- SHORT_CALL 的不利方向：EDB 指向 bullish/up。
- SHORT_PUT 的不利方向：EDB 指向 bearish/down。
- `confidence >= 55`。
- `coverage >= 0.55`。

低 confidence 或低 coverage 不能确认持续性。

### 6.2 GGR Adverse

成立条件之一：

- `regime = NEGATIVE_GAMMA_AMPLIFYING` 且 `abs(distance_to_flip_pct) <= 1.0`。
- GGR veto 为 true。
- pin 不再位于不利边界前方，或 pin trust 明显失效。

GGR 是例外修正项，因为它描述价格接近边界后的非线性加速环境。

### 6.3 KPF Buffer Broken

成立条件之一：

- 不利方向前方无合格厚流动性层。
- 缓冲层已 touched。
- 缓冲层已 penetrated。
- KPF 数据 invalid/expired，且没有 last-good 可用。

KPF 不改变 `touch_probability_now`，只改变 `persistence`。

### 6.4 Persistence Score

```text
0-1 项成立 -> LOW
2 项成立   -> MEDIUM
3 项成立   -> HIGH
```

## 7. 风险分级

### 7.1 Tail Exposure Acceleration

`HIGH`：

- 价格已经触碰或穿透 loss boundary。
- `abs(short_delta) >= 0.72`。
- `short_gamma / entry_short_gamma >= 2.0`。

`MEDIUM`：

- `abs(short_delta) >= 0.55`。
- `short_gamma / entry_short_gamma >= 1.4`。
- 价格距离 loss boundary 已很近但未触碰。

否则为 `LOW`。

### 7.2 Hard Risk

任一成立：

- 已触碰或穿透 loss boundary。
- `touch_probability_now >= 0.78`。
- `touch_probability_now >= 0.62` 且 `touch_probability_drift >= 0.18`。

### 7.3 Rising Risk

任一成立：

- `touch_probability_drift >= 0.12`。
- `recent_deterioration_slope >= 0.22 / hour`。
- `touch_probability_now >= 0.48` 且 `tail_exposure_acceleration != LOW`。

## 8. 摩擦判断

摩擦不决定风险是否存在，只决定动作优先级。

### 8.1 Option Exit Friction

评分：

| 状态 | 分数 |
| --- | ---: |
| LOW / GOOD / CHEAP | 1 |
| MEDIUM / FAIR / NORMAL | 2 |
| HIGH / POOR / WIDE / EXPENSIVE | 3 |
| EXTREME / BLOCKED | 4 |

### 8.2 Hedge Friction Advantage

只有在以下条件成立时，才认为期货对冲优于期权退出：

```text
option_exit_score >= 3
option_exit_score - future_hedge_score >= 1
```

如果期权退出仍可接受，优先退出或减风险，而不是开期货腿。

## 9. 状态机

状态转移按以下顺序判断：

1. 数据缺失、账本不一致、信号包过期、KPF/GGR 关键输入冲突 -> `MANUAL_REVIEW`
2. 已有 hedge leg -> `HEDGE_ACTIVE`
3. `hard_risk` 且 `persistence >= MEDIUM` 且 `hedge_friction_advantage = true` -> `HEDGE_READY`
4. `rising_risk` 且期权退出摩擦可接受 -> `EXIT_PREFERRED`
5. `rising_risk` -> `WATCH`
6. 其他 -> `NORMAL`

解释：

- `HEDGE_READY` 必须最难触发。
- `WATCH` 是“风险变坏但还没有资格对冲”的正式状态。
- `EXIT_PREFERRED` 只在风险已上升且期权退出仍可接受时出现。
- 高触界概率但无持续性确认时，不能直接开期货腿。

## 10. 期货腿方向与尺寸

### 10.1 方向

| 期权结构 | 不利方向 | 期货腿 |
| --- | --- | --- |
| SHORT_CALL | 上涨 | `BUY_FUTURE_OR_PERP` |
| SHORT_PUT | 下跌 | `SELL_FUTURE_OR_PERP` |

### 10.2 尺寸

不追求完全 delta neutral，只压平尾部。

| 档位 | 目标净尾部 delta 降低比例 | 触发 |
| --- | ---: | --- |
| LIGHT | 25%-35% | 人工批准的预对冲或边界接近但未 hard risk |
| MEDIUM | 50%-65% | hard risk 成立但未穿透边界 |
| FULL | 80%-95% | 已触界、p_touch 极高、或 tail acceleration HIGH 且 persistence HIGH |

第一版 `HedgeIntentPackage` 只输出意图，不自动下单。

## 12. 后续实现路线

### Phase 1：文档与干跑

- 记录 `entry_risk_anchor`。
- 每轮生成 `PositionRiskPackage`。
- `HEDGE_READY` 时生成 `HedgeIntentPackage`。
- 只展示，不真实下单。

### Phase 2：回放校验

- 使用真实历史信号包、KPF 包、持仓行情包回放。
- 核查 WATCH、EXIT_PREFERRED、HEDGE_READY 是否符合人工复盘。
- 只允许调整阈值，不扩展新因子。

### Phase 3：小额实盘对冲开关

- 新增独立 `ALLOW_HEDGE_TRADING`。
- 默认 false。
- 与 `ALLOW_TRADING` 分离。
- 仍保留 `DRY_INTENT_ONLY` 面板。

### Phase 4：Adapter 接入

- 先保持 `UNBOUND`。
- 后续根据盘口、资金费率、保证金与运维选择 Deribit 或 Binance。
- adapter 只执行 HedgeIntent，不参与风险判断。

## 13. 验收标准

建议验收时确认：

1. 建仓后账本存在不可覆盖的 `entry_risk_anchor`。
2. 同一输入包可重复生成同一 `PositionRiskPackage`。
3. KPF 不进入触界概率公式。
4. EDB aggregate 是唯一方向证据入口。
5. GGR 只作为非线性加速/例外修正。
6. 期权退出摩擦低时，不生成期货对冲意图。
7. `HEDGE_READY` 必须同时满足风险、持续性和摩擦优势。
8. 第一版不真实下期货单。
9. 所有输出都有 reason_codes。
10. 所有阈值可回放、可审计、可被人工覆盖。

## 14. 本轮结论

基于本轮模拟与当前系统边界，建议路线不是把更多因子塞进概率模型，而是把概率、恶化、持续性、摩擦拆清楚：

- 概率回答“现在离触界有多危险”。
- 恶化回答“这笔仓位是否变坏”。
- 持续性回答“变坏是否有证据继续”。
- 摩擦回答“该退出期权还是临时期货对冲”。

这条路线是当前阶段的最小充分方案建议。它避免了伪精确胜率、KPF 伪预测、EDB 双算、过早期货化，也保留了在真实盘口差时压平尾部的能力。后续是否进入实盘对冲，还需要通过真实回放、小额干跑和独立开关验证。
